### 7. 출결 관리 (`/dashboard/attendance`)

- 요약 pills (전체/출근/지각/미출근)
- **⚠️ 확인이 필요한 교육생** 알림 섹션 (지각/결석자)
- **인라인 수정**: 각 레코드 상태 변경 + 저장 (PATCH API)
- **Excel 업로드** (접이식 📎 버튼):
  - 타임인아웃 SaaS Excel 파일 드래그앤드롭/클릭 업로드
  - `xlsx` 라이브러리로 파싱 (Excel 시리얼 넘버 → 날짜/시간 변환)
  - 업로드 전 미리보기 + 상태 수정 모드
  - 저장 시 이름 → student_id 매핑 후 upsert
- **시간 표시**: `excelSerialToTime()` — 초 단위까지 표시 (08:04:30 형태)
- **note 파싱**: `parseNoteToTimes()` — "출근 08:04:30 / 퇴근 17:30:00" → 출근/퇴근 컬럼 분리
- **API**: `/api/attendance` — GET(조인), POST(upsert), PATCH(수정)

## 삽질 & 해결

| 문제 | 해결 |
|------|------|
| Supabase JS 클라이언트로 raw SQL 실행 불가 | 사용자가 Supabase SQL Editor에서 직접 실행 |
| env 변수 Node 스크립트에서 안 읽힘 | `export $(grep -v '^#' .env.local \| xargs)` 사용 |
| ALTER TABLE 실행 시 "relation already exists" | CREATE TABLE과 분리해서 ALTER TABLE만 실행 |
| 출결 비고 "출근 08:04 / 퇴근 17:30" 비직관적 | `parseNoteToTimes()` 함수로 출근/퇴근 컬럼 분리 |
| 날짜 선택기(input[type=date]) 너무 작음 | CSS min-height 44px + font-size 16px 적용 |
| Tailwind 클래스 Turbopack에서 간헐적 미적용 | 인라인 스타일 + CSS 변수 방식으로 전환 |
