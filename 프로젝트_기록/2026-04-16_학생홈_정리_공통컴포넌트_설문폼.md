# 2026-04-16 — 학생 /my 정리 + 공통 카드/행 컴포넌트 + 설문 폼 개선

## ✅ 오늘 한 것

### 1. 학생 홈 (/my) — 휴무일 / 어제 일지 미작성 알림

**오늘 할 일 카드:**
- 휴무일에는 "출근 체크" 항목 자동 숨김 (`todayDayType !== 'off'` 가드)
- 출결 알림 팝업도 휴무일이면 띄우지 않음
- **어제 일지 미작성 알림** 신규: 어제가 교육일/실습일이었는데 일지 비어있으면 빨간 카드로 "어제 ~ 비어 있어요 / 지금 쓰기 →" 표시
  - `kstYesterday` 계산 시 `toISOString()`이 UTC 변환하면서 하루 더 빠지는 버그 발견 → `T12:00:00Z` + `setUTCDate` 패턴으로 수정

### 2. 공통 카드 컴포넌트 — `<SummaryCard>` 신설

**파일:** `components/SummaryCard.tsx`

기존 교육일지/실습일지/교육설문 카드가 각자 100줄씩 인라인으로 작성되어 있던 걸 하나의 컴포넌트로 통일.

**API 요약:**
- `date` / `typeBadge` / `title` / `titleSize` / `sub` / `thumbnail` / `selected` / `variant` / `onClick` / `href` / `disabled`
- `footerSignals[]`: 칩 배열 (emoji/pill/tag) — **신호 있을 때만** 푸터 노출
- `footerRight`: 자유 ReactNode 또는 `{ type: 'commentCount', count }`

**규격:** padding 24, radius-lg(16), shadow-sm, gap 16, 본문 minHeight 88

**정보 위계 원칙:**
- 3/3, 100% 같은 "다 채움"은 노이즈 → 숨김
- 미완성 신호만 표시 (`참여 1/3`, `STEP 2/3`)
- ⭐ 우수, 💬 코멘트는 강조 유지

**적용 페이지:**
- 교육일지 (`/my/notes`) — 그리드 360px min, 모달 팝업 상세보기, 첫 이미지 썸네일
- 실습일지 (`/my/practice`) — 380px min, 수주 금액을 title로
- 교육설문 (`/my/survey`) — 360px min, 사전/사후 분리

### 3. 공통 행 컴포넌트 — `<SummaryRow>` 신설

**파일:** `components/SummaryRow.tsx`

전체 폭 한 줄짜리 리스트 (공지사항/출결)용. 펼치기 토글 지원.

**API 요약:**
- `leftLabel: { primary, secondary, secondaryTone }` — 좌측 날짜 라벨
- `badge: { text, tone, dot }` — 컬러 닷 prefix 가능
- `title`, `rightSlot` (ReactNode), `expandable` / `expanded` / `onToggle`

**규격:** padding 20×24, CSS Grid 정렬 (날짜 88px / 뱃지 / 제목 1fr / 우측 / 화살표), 날짜 우측에 1px 구분선

**적용 페이지:**
- 공지사항 (`/my/announcements`) — 우선순위 뱃지 + NEW + 시간, 클릭 펼침
- 출결 (`/my/attendance`) — 좌측 날짜/요일 + 출근/지각 컬러닷 뱃지 + 우측 출퇴근 시각, **2열 그리드** (440px min)

### 4. 카드 모달 팝업 패턴

교육일지 카드 클릭 → 인라인 펼침 → **모달 팝업**으로 변경:
- `position: fixed`, 어두운 배경, 외부 클릭 → 닫힘, 모달 자체 스크롤
- 자율학습 모달 배경: `--purple-dim` (반투명) → **솔리드 `--bg-surface` + 보라 2px 테두리**로 변경 (반투명 모달이 뒤 배경 비치는 문제)
- 닫기 버튼: 36×36 정원 강제 (min/max width/height 모두 36, lineHeight: '36px')
- 헤더 우측 액션 버튼들 `paddingRight: 44`로 닫기 버튼 자리 확보 + flex-wrap 허용

### 5. 교육설문 페이지 정리

**카드 단계 (`/my/survey`):**
- 자체 padding/maxWidth 제거 → layout.tsx 기본 사용 (위치/여백 통일)
- h1 clamp(28~40px) → h2 28px (다른 페이지와 통일)
- 각 카드에 응답 기간(`period`) 추가: "4/15 ~ 4/15 자정"
- 사전/사후 카드 클릭 시 `?phase=pre|post` query param으로 분리 — 양쪽 동시 노출 X

**상세 페이지 (`/my/survey/ansan-tour`):**
- 자체 padding/maxWidth 720 제거 → 다른 페이지와 동일 레이아웃
- 페이지 상단 중복 제목 ("안성공장 인프라 투어 설문" + "안성공장 투어 — 사전 설문") → 한 줄로 통합
- query param phase에 따라 사전/사후 한 단계만 표시
- **마감일 체크**: `SURVEY_DEADLINES` 상수 (`pre/post: '2026-04-15'`) 기준, 마감되면 "수정하기" 버튼 → "마감" 회색 뱃지로 교체
- "best_line" (가장 인상 깊은 라인) 답변 표시 — 읽기 전용 모드에서는 선택된 옵션만 파란 알약으로 표시 (빈 값이면 "선택 안 함")

**설문 폼 자체:**
- **2열 그리드 분리**: 왼쪽=객관식(점수), 오른쪽=주관식(텍스트)
  - 사전: 좌(자가진단 9문항) / 우(호기심 5문항)
  - 사후: 좌(자가진단 + 만족도/NPS) / 우(인상 깊은 라인 + 새로 알게 된 점)
- **질문 위계 강화**: Q번호 뱃지(파란 원 1,2,3...) + 굵게 + 질문 사이 구분선 + 답변 영역 들여쓰기 34px
- **5점 척도 버튼 통일 크기**: `grid 5등분`으로 정확히 같은 너비, 버튼 안에는 숫자만, 양 끝에만 "전혀 모름"/"매우 잘 앎" 라벨 한 번

## 🤔 결정한 것

- **카드 그리드 컴포넌트와 행 리스트 컴포넌트는 분리** (SummaryCard / SummaryRow). 같은 톤 enum을 공유하되 레이아웃 책임은 각자.
- **푸터 신호는 "있을 때만"**: 3/3 같은 완료 표시는 정보 가치가 없으므로 노이즈로 간주하고 숨김
- **카드 클릭 → 모달 팝업**: 인라인 펼침은 카드 위치가 흐트러지므로 중앙 모달이 더 자연스러움
- **모달 배경은 무조건 솔리드**: 변형(자율학습 등)이 있어도 반투명 배경 사용 X, 테두리로 구분
- **주차/단계 선택은 밑줄 탭이 표준**: 둥근 알약 X (DESIGN_SYSTEM §8 + Anti-pattern 추가)
- **모바일에서 탭 → 드롭다운으로 전환**: 768px 이하면 `<select>` 사용
- **설문 답변 기간/마감일은 카드와 상세 페이지 모두에 표시**: 학생이 마감 인지 가능하도록
- **마감 후 수정 불가**: 마감일 지나면 "수정하기" 버튼 자체를 회색 "마감" 뱃지로 대체
