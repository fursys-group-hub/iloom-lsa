### 4) 자신감 분포 차트 KST 버킷팅 버그 수정
- **증상**: `/dashboard/analytics` → "자신감 분포 변화" 차트에서 3/29 칸만 빈 바로 표시됨 (주말인데 왜 보이나?)
- **원인**: `AnalyticsClient.tsx`의 `noteAnalysis` 집계가 `n.created_at.slice(0, 10)` 로 **UTC 날짜** 기준 버킷팅
  - 김창석의 교육일지 created_at이 `UTC 2026-03-29 23:32` (KST로는 3/30 08:32) 이어서 3/29 칸에 잘못 분류됨
  - 그 노트의 confidence가 null이라 모든 비율 0% → 빈 바
- **수정**: `new Date(n.created_at).toLocaleDateString('sv-SE', { timeZone: 'Asia/Seoul' })` 로 KST 기준 버킷팅
- 자신감 분포 + 참여도 두 차트가 같은 `eduNotes`에서 파생되므로 한 줄 수정으로 둘 다 해결됨
- **커밋**: `fabc029` — "가라 데이터 정리 + 자신감 분포 차트 KST 버킷팅 수정"

### 🔥 우선순위 1 — 교육 인사이트 페이지의 수주 전환율 기반 데이터 소스 재설계

**영향받는 3개 위치** (`app/dashboard/analytics/AnalyticsClient.tsx`):

1. **"교육이 현장 성과로 이어졌나?" 섹션** (874~946줄)
   - 전체 퍼널 차트에서 "실습 N / 심화 N" 비교 → 심화만 표시
   - 수주 금액도 "실습 X원 / 심화 Y원" → 심화만
   - WhyPopover 설명문에서 실습일지 설명 제거
   - `sectionSub` "실습일지 + 심화교육 주간 수주 데이터 기반" → "심화교육 주간 수주 데이터 기반"

2. **"시험 성적 vs 수주 전환율" 산점도** (954~1017줄)
   - `pData + wData` 합산 → `wData`만
   - 조건을 `batchWeeklySales.length > 0`로

3. **"교육 지표 vs 현장 성과" 테이블 + 상관계수** (1019~1074줄)
   - `수주전환율` 컬럼: `practiceData` 제거
   - 상관계수 계산에서도 `pData` 제거

**현재 상태**: weekly_sales는 이미 다 삭제됨 → 수정 후 카드들이 "심화교육 수주 데이터 업로드 전" 빈 상태로 보이게 됨. 나중에 심화 엑셀 업로드하면 자동으로 채워짐

- **옵션 A**: 실습일지 완전 제외. 실습일지 분석은 실습일지 페이지에서만 봄 (깔끔하지만 교육 효과 증거가 심화 수주 업로드 전까진 아예 없음)
