## ✅ 오늘 한 것

### 1. 두께 섹션 — 사진 옆 텍스트 구조 정돈
원본은 사진 2개 가로 + 큰 stat 박스 + 캡션이 따로 떠 있는 형태였음. 사용자 피드백("뭔가 중구난방") → 사진 좌·텍스트 우 매칭 행 구조(`.thickness-rows` / `.thickness-row`)로 1차 정돈.

### 2. 좌측 사진 인디케이터 + 행 사이 연결고리
- 사진 위쪽에 검정 배지 + ▼ 화살표(`.photo-pointer-label`) — `매일 이 자리에 짐이 쌓여요`
- 우측 텍스트는 인용문 톤(`.row-question` 12pt 굵게)으로 격상 — `"이만큼 짐 받으면 윗판이 처지지 않나요?"`
- 두 행 사이 연결선(`.thickness-row-bridge`) — `그래서 우리는 천판부터 다르게 만들어요`

### 3. 이불·박스·여행가방 이모지 칩
- 사진을 이모지로 대체하는 안은 거절(브랜드 톤·인쇄 안정성 우려) → 사진은 유지하고 칩으로 보조
- `.load-chips` / `.load-chip` 알약 칩 디자인 — 🛏️ 이불 / 📦 박스 / 🧳 여행가방
- 우측 질문 텍스트 위에 한 줄 배치

### 4. 좌우 2-컬럼 비교 패널 (문제↔해결)
- 위→아래 2행 → 좌→우 2컬럼 비교로 재구성 (`.problem-solution`)
- 좌측 사진 흑백 처리(`filter: grayscale(100%) contrast(0.96) brightness(0.98)`) — "문제"/"이슈" 시각 신호
- 좌측 = 자주 듣는 질문 / 우측 = 일룸 컬렉트의 답 (pill 태그)

### 5. material-flow 카드 패턴으로 전환 (재정돈)
사용자 피드백("이렇게 디자인된 거 말고 그냥 위처럼 정의된 스타일") → 위쪽 LPM/PET 카드의 `.flow-side` 구조와 100% 동일하게 통일:
- 흰 카드 + 얇은 테두리 + 사진 상단 + `flow-body` 본문
- pill 태그(`자주 듣는 질문`/`일룸 컬렉트의 답`) 삭제
- 본문 최상단에 `.flow-side-name` 제목(`고객분들께 자주 듣는 질문` / `일룸의 해결`) — 자동으로 구분선 깔림
- 칩/큰 stat(`+0.5mm → 6~8%`)/styled 인용구 모두 제거
- 모든 콘텐츠는 `.flow-list` 불릿으로 흡수 (LPM 카드와 시각 톤 완전 동일)

### 6. material-flow(LPM/PET) 카드 폭 + 부제 정리
- `.material-flow` 그리드 `310px 310px` 고정 → `1fr 1fr` 가변(부모 폭 꽉 채움)
- 부제 두 줄 삭제 — `열압 일체화 — 한 몸으로 압착` / `냉압 본드 부착 — 종이/필름을 본드로 붙임`
- `.flow-side-tag`가 갖고 있던 구분선(border-bottom + padding-bottom)을 `.flow-side-name`으로 이전

### 7. 좌측 사진 위치 조정
원본 이미지의 중간이 잡혀서 베개만 보임 → `object-position: center bottom` 적용해 하단부 사용.

### 8. 우측 카드 불릿 — `—` → `→` 화살표 + 줄바꿈 고정
- em 대시(`—`)가 잘 안 보이고 줄바꿈이 어색 → `→` 화살표로 교체
- 화살표 직후 `<br>` 삽입 — 1줄 끝에 화살표 고정, 2줄에 결과 (`원인 → \n 결과` 시각 흐름)

### 9. 측판 narrative 한 줄 제거
페이지 하단 `측판은 한샘 베이직이 더 두꺼워 보이지만 코팅페이퍼라...`는 PAGE 3(LPM vs 코팅페이퍼)의 불릿(`본드가 약해지면 결국 들뜸`, `옷걸이 후크 스침에 미세 흠집 누적`)과 내용 중복 → 두께 페이지에서는 삭제. 두께 페이지가 "두께/강도" 한 가지 메시지에 집중되도록.

### 10. pitch-line 두께 메시지로 재작성
원본은 표면재(코팅페이퍼/LPM)와 두께가 섞여 있었음 → 두께 전용으로 재작성:
> "한샘 시그니처가 윗판 18mm·옆판 15mm인데 우리는 *윗판 18.5mm·옆판 15.5mm*예요. 0.5mm 차이가 작아 보여도 매일 짐을 받는 윗판은 *휨 강도가 6~8% 우위*라, *10년 사용해도 처지지 않고 거뜬*합니다."

흐름: 비교 수치 → 고객 의문 선제 답변(0.5mm 작아 보여도) → 결과(6~8% 우위 → 10년 거뜬).

### 11. 고객 친화어 전체 통일
**핵심 결정** — 천판/측판/지판은 가구 업계 용어라 일반 고객(입주박람회 방문객)이 모름. 영업현장 톤 살리려고 PAGE 2/4/5 전체에서 일괄 치환:
- 천판 → **윗판**
- 측판 → **옆판**
- 지판 → **바닥판**

| 페이지 | 변경 위치 |
|---|---|
| PAGE 2 (한눈에 비교) | 종합 비교표 행: `천판/지판 두께`, `측판 두께` |
| PAGE 4 (두께) | 인트로, 카드 제목, 비교표, 카드 불릿 6개, 사진 라벨, pitch-line (총 9곳) |

## 🤔 결정한 것

1. **사진 vs 이모지** — 박람회 자료에서 사진을 이모지로 대체하는 건 위험(브랜드 톤·인쇄 폰트 불안정·신뢰도 ↓). 사진은 유지하고 이모지는 보조 칩으로만 사용. 이후 LPM 패턴 통일 단계에서 칩도 제거됨.
2. **막대그래프 비교 안 채택** — `+0.5mm → 6~8%`를 18.0mm vs 18.5mm 막대 비교로 시각화하는 안은 "예쁘지 않다"는 피드백으로 거절. 대신 큰 stat 헤드라인 → 최종적으로 `.flow-list` 불릿으로 흡수.
3. **카드 디자인은 기존 패턴 재사용** — 새 디자인 클래스(`.problem-solution`, `.ps-panel`, `.ps-panel-tag`) 만들었다가 사용자가 "위(LPM) 정의된 스타일 그대로"를 요구해 모두 폐기하고 `.flow-side` 통일. **교훈: 문서 안에 이미 검증된 패턴이 있으면 그걸 먼저 재사용**.
4. **두께 ↔ 표면재 책임 분리** — PAGE 3 = 표면재(LPM vs 코팅페이퍼) / PAGE 4 = 두께. 페이지마다 한 가지 메시지에만 집중 → 측판 narrative 같은 cross-topic 콘텐츠는 제거.
5. **고객 친화어 통일은 영업톤 현장적용 우선** — 가구 업계 용어 정확성보다 박람회 고객이 즉각 이해하는 게 더 중요. 윗판/옆판/바닥판으로 전면 통일.
