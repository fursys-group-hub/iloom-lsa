# 2026-04-19 — 전역 기수 컨텍스트(BatchContext) 도입

## ✅ 오늘 한 것

### 발단
교육일지 화면에서 **미제출자 명단에 어제 임포트한 테스트 학생(테스트1~5)이 그대로 남아있는 버그** 발견.
원인 점검: [education-logs/page.tsx](../app/dashboard/education-logs/page.tsx)가 `/api/students`를 호출하면서 `batch_id` 파라미터를 안 넘김 → **모든 기수의 모든 학생을 미제출 모수에 포함**.

### 전수 점검 결과 — 7개 페이지에서 같은 버그 확인

| 페이지 | 문제 |
|--------|------|
| 교육일지 | 미제출 명단에 다른 기수 학생 포함 |
| 실습일지 | 날짜 드롭다운에 여러 기수 기록 혼재 |
| 출결 | 인원수가 모든 기수 합쳐짐 |
| 테스트 | `batches[0]`로 고정, 기수 전환 UI 없음 |
| 질문관리 | 다른 기수 학생의 질문이 답변 대기로 섞임 |
| 심화교육 평가 | 매장 관리자 평가에 모든 기수 노출 |
| 심화교육 오버뷰 | 통계/차트가 다중 기수 평균 |

홈 대시보드만 자체 기수 드롭다운으로 정상 작동.

### 채택한 방향: B안 — 글로벌 기수 컨텍스트
페이지마다 드롭다운을 추가하는 A안보다, **사이드바에 전역 드롭다운 1개**를 두고 모든 페이지가 공유하는 구조가 합리적이라고 판단.
- 7번 중복 코드 제거
- 페이지 이동 시 기수 재선택 불필요 (UX 개선)
- 앞으로 신규 페이지가 추가돼도 누락 위험 없음

### 핵심 설계

`student_notes`, `attendance`, `student_questions`, `weekly_evaluations` 등은 **`batch_id` 컬럼이 없고 `student_id`만 있음**. 그래서 다음 패턴으로 통일:

```ts
const { selectedBatchId } = useBatch();

const fetchData = useCallback(async () => {
  if (!selectedBatchId) return;
  const [dataRes, studentsRes] = await Promise.all([
    fetch('/api/notes?all=true'),
    fetch(`/api/students?batch_id=${selectedBatchId}`),
  ]);
  const ids = new Set(students.map(s => s.id));
  const filtered = data.filter(n => ids.has(n.student_id));
  // ...
}, [selectedBatchId]);
```

API는 손대지 않음 (`/api/students?batch_id=`만 활용 — 이미 지원 중).

## 🤔 결정한 것

- **글로벌 BatchContext 채택** — 페이지마다 드롭다운 박는 A안 대신
- **API 변경 없이 클라이언트 필터링** — `student_notes` 등에 `batch_id` 컬럼이 없는 현 스키마 유지
- **드롭다운 위치는 사이드바 로고 아래** — 메인 헤더보다 사이드바가 더 일관성 있는 영역
- **localStorage 키: `iloom-selected-batch`** — 기존 `iloom-auth`, `iloom-theme`와 동일한 prefix 컨벤션
- **기본 기수 선택 우선순위**: 진행중(입문/심화) → 가장 최근 활성 → 첫 기수
- **홈 우상단 자체 드롭다운 제거** — 사이드바 드롭다운 하나로 통일
