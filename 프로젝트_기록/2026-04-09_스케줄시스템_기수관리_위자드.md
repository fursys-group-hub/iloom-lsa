# 2026-04-09 — 스케줄 시스템 고도화 + 기수관리 마법사 + 교육생 안내 UX

## ✅ 오늘 한 것 (2차 세션):

### 1. 스케줄 시스템 날짜 수정 + 검증
- 4/9가 정규교육으로 잘못 저장된 것 발견 → 4/8까지 정규교육, 4/9부터 휴무로 수정
- 전체 날짜 검증 완료 (4/8~4/22 전부 ✅)
- 최종 스케줄: 정규교육 19일 / 매장실습 4일 / 휴무 8일

### 2. 홈 대시보드 스케줄 반영
- "오늘 교육일지" 카드에 스케줄 연동
- 휴무일: "🌙 오늘은 휴무일이에요. 교육일지 제출이 필요하지 않아요!" (빨간 미제출 목록 안 나옴)
- 매장실습일: "🏪 오늘은 매장실습일이에요. 실습일지를 확인해주세요!" + 실습일지 링크
- 정규교육일: 기존 제출/미제출 카드 그대로
- 날짜 유형 뱃지 표시 (📚정규교육 / 🏪매장실습 / 🌙휴무)

### 3. 교육생 화면 스케줄 안내
- **교육생 홈(/my)**: 로그인 시 오늘 일정에 맞는 배너 자동 표시
  - 휴무일: 🌙 "오늘은 휴무일이에요!" + 자율학습 쓰기 버튼 (보라)
  - 매장실습일: 🏪 "오늘은 매장실습일이에요! 실습일지 작성이 필요해요" + 실습일지 버튼 (주황)
- **교육일지(/my/notes)**: 상단에 휴무/실습 안내 배너
  - 휴무일: "교육일지 제출이 필요 없어요. 작성하면 자율학습으로 저장돼요"
  - 매장실습일: "실습일지를 작성해주세요" + 실습일지 → 링크

### 4. 질문관리 보관처리 DB 수정
- `student_questions.status` check constraint에 'archived' 추가
- SQL: `ALTER TABLE student_questions DROP CONSTRAINT ... ADD CONSTRAINT ... CHECK (status IN ('open', 'answered', 'resolved', 'archived'))`
- 보관 버튼 정상 동작 확인

### 5. 기수관리 3단계 마법사(wizard) UI
- 기존 한 페이지 폼 → **3단계 wizard** 구조로 개편
- **STEP 1**: 기수명 + 입문교육 기간 입력 → "다음: 스케줄 설정"
- **STEP 2**: 드래그 달력으로 정규교육/매장실습/휴무 칠하기
  - 페인트 모드 3개 (📚/🏪/🌙) 선택 후 달력 드래그
  - 월간 달력 그리드 (일~토 7열), 월별 분리
  - 새 기수: 평일=정규교육, 주말=휴무 자동 생성
  - 수정 시: 기존 스케줄 로드
  - 합계: 📚19 🏪4 🌙8 총31일
- **STEP 3**: 심화교육 일정 (선택, 나중에 설정 가능) → "등록 완료"
- 스텝 인디케이터 클릭으로 이전 단계 이동 가능
