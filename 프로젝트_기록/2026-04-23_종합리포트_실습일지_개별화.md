### 2. `scripts/generate-reports.js` 2가지 개선
1. **`.insert()` → `.upsert()`** — `onConflict: 'student_id,test_date,report_type'` 적용. 같은 날 재실행 시 덮어쓰기 동작 (기존에는 중복 저장 에러 가능성).
2. **`practice-rewrites.json` 우선 참조 로직 추가** — 기존 `synthesizePractice()` 템플릿은 모든 학생에게 "수주 있으면 → 기본 역량이 갖춰져 있음" 같은 공통 문장을 박아넣어 개별성이 없었음. JSON에 학생명이 있으면 커스텀 서술을, 없으면 기존 템플릿으로 fallback.

### 3. `scripts/practice-rewrites.json` 신설 (12명 개별화)
각 학생의 실습일지 4일치 step1~4 원문을 직접 읽고 **생활기록부 톤으로 개별 관찰 문장 작성**. 실제 일지 구절을 직접 인용하여 학생 고유 성향이 드러나도록 함.

예시:
- **최시원**: "==자기비하 반복== ('말하는 감자', '더 짱큰 명찰')... 매트리스·쿠시노 영역에서 **고객의 반감을 뒤집어 구매로 전환**한 성공 사례"
- **박정인**: "첫날 '일빠' 역할 자원... ==전도방지밴드 6천 원을 고객 의사 없이 자의적으로 넣은 점== 반성... '나약한 마음으론 정규직 어림도 없겠다' 성취 동기"
- **김강욱**: "==일지 분량이 짧고 단문 위주==... 혼자 견적서를 복기하며 다시 작성... '퉁명스럽게 응대하시던 부부가 밝은 미소로 재방문 약속'"
- **안덕찬**: "상담 건별로 10건 이상, 각 건마다 '상황-대응-아쉬운 점'을 구조적으로 서술할 만큼 ==문서화·자기성찰 역량이 동기 중 최상위권=="

### 4. `scripts/memo-rewrites.json` 5명 추가
교육자 메모(`student_memos`)가 DB에 비어있는 상태에서, 실습일지의 자기인식·태도 데이터를 기반으로 누락된 5명(나원빈/서재현/이예나/채형우/최시원)의 교육자 시점 서술을 생활기록부 톤으로 추가.

## 🤔 결정한 것

1. **템플릿 자동화 vs 개별 리라이팅** — 초기 실행 후 사용자가 "다 너무 똑같다"고 피드백 → 12명 × 4일 실습일지를 직접 읽고 학생별 관찰 문장 수동 작성하는 A안 채택. 다음 기수부터는 `practice-rewrites.json`만 새로 채우면 동일 품질 재사용 가능.
2. **테스트 계정 처리** — 사용자가 프롬프트 목록에 테스트1~5를 포함했지만 students 테이블에 없어 자동 제외. batch_id 기반 필터링으로 충돌 없음.
3. **재실행 안전성** — `.upsert()`로 변경해서 오늘자 그룹 2회 재생성해도 문제 없음. (이번 세션에서 실제로 2회 재생성: 초기 → 개별화)
