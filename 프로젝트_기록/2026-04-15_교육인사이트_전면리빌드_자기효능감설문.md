# 2026-04-15 — 교육 인사이트 페이지 전면 리빌드 + 자기효능감 설문

## 배경

기존 교육 인사이트 페이지가 **가상(mock) 수주 데이터**로 교차 분석을 보여주고 있어 실질적인 의미가 없었음.
HR 관점에서 "이 교육이 진짜 효과 있나?"를 증명할 수 있는 페이지로 전면 리빌드.

---

## ✅ 오늘 한 것

### 1. DB 스키마 2개 테이블 신규 생성

**`education_surveys`** — 자기효능감 + 만족도 통합 설문 (사전-사후 2회)
- 자기효능감 5문항 (제품지식/고객응대/판매성사/팀워크/전반적준비도)
- 만족도 3문항 (교육내용/방식/기간)
- 주관식 3문항 (성장/걱정/목표)
- phase 구분: `intro_end` (입문 끝) / `advanced_end` (심화 끝)
- UNIQUE: `(batch_id, student_id, phase)` — 같은 학생이 시점별 1회씩

**`weekly_sales`** — 심화교육 주간 수주 실적
- 1~12주차까지 주차별 저장
- 상담/견적/수주 건수 + 수주금액 + 카테고리(배열) + 메모
- 엑셀 업로드로 일괄 입력

### 2. API 엔드포인트 신규 2개

| 파일 | 역할 |
|------|------|
| `/api/education-surveys` | GET/POST (upsert) — 설문 조회 + 저장 |
| `/api/weekly-sales` | GET/POST(JSON+엑셀)/DELETE — 주간 수주 CRUD + XLSX 파싱 |

엑셀 업로드 형식: `교육생명 | 주차 | 상담 | 견적 | 수주 | 금액 | 카테고리 | 메모`
학생명 → ID 매핑 + 주차 유효성(1~12) 자동 검증.

### 3. 교육생용 설문 페이지 `/my/survey`

- 한 페이지에 3파트: 자신감(5점 척도) + 만족도(5점 척도) + 주관식(3개 텍스트)
- phase 자동 감지: 입문 끝 설문 없으면 `intro_end` 폼, 있으면 `advanced_end`
- 기존 응답 수정 가능 (수정 버튼)
- **HR 전문 용어 숨김**: "자기효능감" → "지금 나의 자신감은?", "교육 만족도" → "교육은 어땠나요?"
  - 이유: 설문 의도가 노출되면 솔직한 답변이 안 나옴
- 사이드바 메뉴 추가 (`/my/layout.tsx`)
- 5점 척도 버튼: `radius-sm`(8px) — 너무 동그란 pill 느낌 해소

### 4. 교육 인사이트 페이지 전면 리빌드

**제거된 것:**
- `generateMockSalesData()` — 가상 수주 생성 함수
- 4개 mock 교차 분석 섹션 (crossScoreVsSales 외 3개)
- mock 경고 배너
- "변화 배너 +1점" 카드 (KPI 카드와 중복이라 제거)

**새 구조 (7개 섹션):**

| 섹션 | 내용 | 차트 |
|------|------|------|
| 0. KPI | 평균 시험/적응/출석률/자기효능감 변화 (4칸 한 카드) | 숫자 + 뱃지 |
| 1. 자기효능감 + 만족도 + 목소리 | 한 카드 안 좌:우 4:6 분할 + 가운데 구분선 | **레이더(오각형) + 표** |
| 2. 자신감/참여도 변화 | 교육일지 기반 추이 (2열) | 스택바 + 라인 |
| 3. 시험 추이 + 적응 분포 | 지식 성장 증거 (2열) | 라인 + 3그룹 카드 |
| 4. 분야별 효과 + 키워드 | 카테고리 정답률 + 실습일지 키워드 (2열) | 가로바 + 태그 |
| 5. 현장 성과 | 실습 vs 심화 퍼널 + 교육생별 수주 성장 | 퍼널 + **멀티라인 밴드** |
| 6. 교차분석 + 다음 기수 제안 + 업로드 | 시험↔수주 상관관계 + 엑셀 업로드 | **산점도 + Pearson r** |

### 5. HR 친화적 인포그래픽 패턴

**각 카드 구조:**
```
[큰 헤드라인 — 사람 말투] "교육 후 [영역] 자신감이 가장 많이 자랐어요"
                                                          [왜 보나요?]
[작은 데이터 근거] "+1.2점 성장 · 자신감 사전-사후 (5점 만점)"
[메인 차트/표]
[하이라이트 인사이트 박스]
```

- 헤드라인: 22px / 800 / line-height 1.2 / 3줄 줄바꿈 패턴
- 데이터: 13px / muted 보조 텍스트
- "왜 보나요?" popover: 12px / 400 / muted — 클릭하면 상세 설명 (5~10줄)

### 6. "왜 보나요?" Popover 재사용 컴포넌트

`WhyPopover` 컴포넌트로 전 카드에 일관성 있게 적용:
- KPI 4가지 지표
- 자기효능감 / 만족도
- 자신감 분포 / 참여 깊이
- 시험 추이 / 적응 분포
- 분야별 효과 / 실습 키워드
- 현장 성과 / 시험↔수주 상관관계

각 popover에는 **맥락, 측정 방식, 해석, 활용 방법, 커크패트릭 매핑** 포함 (비개발자/HR 담당자가 이해할 수 있도록 자세하게).

### 7. HR 측정 프레임워크

**커크패트릭 4단계** 매핑:
| 단계 | 측정 | 도구 |
|------|------|------|
| 1. 반응 | 교육 만족도 | 설문 (입문 끝) |
| 2. 학습 | 지식 습득 | 시험 + 적응지수 |
| 3. 행동 | 현장 적용 | 자기효능감 설문 사전-사후 |
| 4. 성과 | 비즈니스 결과 | 주간 수주 엑셀 |

### 8. 교육생 상세 페이지 "성장 여정" 섹션 추가

`/dashboard/students/[id]`에 IDP(Individual Development Plan) 섹션:
- 자기효능감 사전-사후 미니 바 차트
- 자신감 타임라인 (교육일지 이모지)
- 참여도 변화 (초반 3일 vs 후반 3일)
- 주관식 하이라이트 (성장/걱정)
- 모바일 순서: 적응지수 바로 다음 (order: 2)

### 9. 차트 다양화 (HR 친화적)

| 차트 타입 | 사용처 | 이유 |
|----------|--------|------|
| **레이더 (polygon)** | 자기효능감 5영역 | 영역별 성장을 한눈에 |
| **산점도 (ScatterChart)** | 시험 vs 수주 전환율 | 우상향 = 상관관계 직관 |
| **멀티라인 밴드** | 교육생별 주차 수주 | 누가 성장/정체인지 비교 |
| **스택바** | 자신감 분포 변화 | 비율 변화 추이 |

### 10. 모바일 반응형 정리

## 🤔 결정한 것

- **같은 설문 데이터는 한 카드에 묶기**: 자기효능감 + 만족도 + 목소리를 별도 카드 3개가 아닌 **하나의 콤보 카드**에. 같은 시점(설문)에서 나온 데이터이므로 분리하면 맥락 끊김
- **위험 교육생 발견은 인사이트 페이지에 넣지 않음**: 교육생 분석 페이지에 이미 있으므로 중복 제거
- **페이지 목적은 "영업 대시보드"가 아니라 "교육 효과 증명"**: Q1(효과 있었나?) + Q3(다음 기수 개선) 두 가지에 집중
- **가상 수주 유지 vs 실제 데이터 교체**: 실습일지에 이미 상담/견적/수주 데이터가 있으므로 mock 완전 제거
- **HR 전문 용어는 숨기기**: 교육생 설문에서 "자기효능감" 같은 용어는 의도 노출 → 자연스러운 한국어로
- **인포그래픽 스타일**: WeTransfer 스타일의 "스토리 헤드라인 + 데이터 근거" 패턴이 HR에게 가장 효과적
