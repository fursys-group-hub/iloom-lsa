## ✅ 오늘 한 것

### 1. 출결 자동화 (타임인아웃 연동)

#### Playwright 스크립트 생성
- `scripts/attendance/fetch-attendance.js` — 메인 실행 (로그인→다운로드→파싱→API전송)
- `scripts/attendance/timeinout-auth.js` — timeinout.kr 자동 로그인
- `scripts/attendance/timeinout-scraper.js` — 출퇴근현황 Excel 다운로드
- `scripts/attendance/run-attendance.bat` — Windows 작업 스케줄러용 배치 파일
- `lib/attendance-parser.ts` — 출결 파싱 로직 공유 모듈 (기존 page.tsx에서 추출)

#### 실행 방법
```bash
npm run attendance              # 헤드리스 모드
npm run attendance:headed       # 브라우저 보이는 모드 (디버깅)
```

- 출근 체크 전까지 페이지 방문 시 반복 표시
- 30분마다 자동 재확인 → 출근 완료 시 자동 해제

### 2. 시험 자동 동기화 Cron

#### 구현
- `lib/sync.ts` — 기존 `/api/sync` POST 핸들러에서 핵심 로직 추출 (`syncBatch()` 함수)
- `app/api/cron/sync/route.ts` — Cron 전용 GET 엔드포인트
  - `Authorization: Bearer CRON_SECRET` 헤더 인증
  - 진행중인 기수(입문/심화 교육 기간) 자동 판별
  - sheet_id 있는 기수만 `new_only` 모드로 동기화

#### 설정
- Railway 환경변수: `CRON_SECRET=iloom-cron-sync-2026-secret`
- cron-job.org: 매일 09:30 KST → `GET /api/cron/sync` 호출

### 3. 기수 아카이브

#### DB 스키마
```sql

ALTER TABLE batches ADD COLUMN archived_at TIMESTAMPTZ;
```

#### 타입 업데이트
- `lib/types.ts` — Batch 인터페이스에 `is_archived`, `archived_at`, `advanced_start`, `advanced_end` 추가
- `app/dashboard/DashboardClient.tsx` — BatchInfo에 `is_archived` 추가
- `app/dashboard/settings/page.tsx` — Batch에 `is_archived`, `archived_at` 추가

#### 아카이브 체크 헬퍼
- `lib/archive-check.ts` — `isBatchArchived(supabase, studentId)` 함수

#### API 수정
- `app/api/batches/route.ts` — PATCH에서 아카이브/복구 처리
- `app/api/auth/route.ts` — 로그인 응답에 `isArchived` 플래그 추가
- 학생 쓰기 API 4개에 아카이브 체크 → 403 차단:
  - `app/api/notes/route.ts`
  - `app/api/student-questions/route.ts`
  - `app/api/note-comments/route.ts`
  - `app/api/benchmarks/route.ts`

#### UI 수정
- **기수관리**: 완료된 기수에 "📦 보관" 버튼, 아카이브된 기수에 "복구" 버튼
- **대시보드 드롭다운**: 아카이브 기수 하단 분리 + "(보관)" 표시
- **학생 홈**: 아카이브 시 📦 읽기전용 배너
