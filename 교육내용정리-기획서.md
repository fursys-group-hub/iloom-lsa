# 📚 교육내용정리 시스템 — 기획서

> **시작일**: 2026-04-27
> **상태**: 1차 인프라 완성 (코펜하겐 기준 모델 + 자동 카탈로그 수집)
> **목적**: 교육생들이 작성한 일지 + 일룸 공식 자료를 통합해 다음 기수 교재 자동 생성
> **위치**: `1) 입문교육` 프로젝트 안 (Next.js + Supabase 동일 인프라 활용)

## 🎯 프로젝트 배경

### 왜 만들었나
수지님이 신입 영업사원 입문교육 교재를 직접 손으로 정리해왔는데, 다음 한계가 있었음:
- **단종된 제품 정보 누락** — 교재가 오래되면 반영 어려움
- **사양 변경 미반영** — 색상 추가, 모델 변경 등을 일일이 못 따라감
- **현장 노하우 부재** — 매장 실습에서만 알 수 있는 영업 팁/고객 사례가 빠짐

한편 교육생들이 매일 작성하는 **교육일지**와 **자율학습 일지**에는 매장에서 직접 들은 디테일과 본인이 이해한 핵심 내용이 누적되어 있음. 이걸 활용해 살아있는 교재를 만들자는 발상.

### 결과물 형태
- **HTML 기반** → **PDF 인쇄** (16:9 가로 슬라이드, 1920×1080 비율)
- **시리즈별 1챕터** (예: 코펜하겐 챕터, 뉴트 챕터)
- **신입이 매장에 가져갈 수 있는 영업 가이드북**

## 🧭 사용자 결정 사항 (모든 피드백 누적)

작업 중 수지님이 직접 결정한 사항들. 다음 작업 시 이 결정을 그대로 따라야 함.

### 데이터 소스
1. **교육일지** + **자율학습 일지** 사용 (실습일지 제외 — 해석 패턴 다름)
2. **모든 기수 누적** (P3, P5 등)
3. **실습일지 제외** — 의도적
4. **온라인 전용 제품**은 별도 표기 (가격/제품 코드 다름)

### 분류 방식
5. **AI(Gemini)가 아닌 정규식 매칭**으로 일지 분류 — 5분 → 5초 단축
6. 분류는 시리즈명(94개) 기반, 일지 본문에서 직접 추출
7. **카탈로그(일룸 가이드 사이트) 기반**으로 메인 페이지 시리즈 목록 표시 (`series-map.ts` 정적 매핑 폐기)

### AI 작성 정책
8. **Gemini 사용 안 함** — 응답 부정확, 시간 오래 걸림
9. **Claude(Code)가 직접 일지 분석**해서 영업 노하우 작성 — AI 코칭 리포트 패턴과 동일
10. 사양 정보(코드/사이즈/색상)는 **자동화 스크립트**로
11. 영업 노하우(특장점/매장 팁/깨달음)는 **Claude가 일지 읽고 작성**

### 챕터 구조 (코펜하겐 = 기준 모델)
12. **섹션 순서**: 기본 정보 → 제품 특장점 → 매장 실전 팁 → 학생 깨달음 → (구분선) → 운영 라인업 → 색상 옵션 → 소재 → 사이즈 도면
13. 사양 표는 뒤로, **영업 노하우(왜 이 제품이 좋은지)** 앞으로
14. **"왜 그게 개방감인가"처럼 인과관계 풀어 설명** — 키워드만 나열 금지
15. **추천 고객 페르소나** + **마감재별 추천** 표
16. **영업 멘트 예시** 노란 박스로 강조
17. **단종/사양변경 알림 빈 섹션은 표시 안 함** (정보 있을 때만)
18. **출처 노트(WordPress + PPT + 일지 통합 같은) 숨김**

### 디자인 표준
19. **DESIGN_SYSTEM.md 준수** (뱃지 fontSize 12, fontWeight 600, padding 3px 10px)
20. **색상 칩 표 형식**: 가로 배치 (한 줄에 5색) + 한국어/영문 한 셀에 줄바꿈으로
21. **사진 placeholder** 자리 (수동 첨부) — 나중에 실제 이미지로 교체
22. **카드 디자인**: 깔끔한 책 느낌, 흰 바탕, Pretendard 폰트, 인쇄 친화적
23. **매장 영업 필수 안내**는 **빨간 박스로 강조** (브라켓 방식 같은 영업 시 클레임 예방용)

### PDF 인쇄
24. **16:9 가로 슬라이드** (PowerPoint 표준 13.333×7.5인치 = 1920×1080 비율)
25. **섹션마다 새 페이지** (h2 기준 page-break-before)
26. **사이드바 인쇄 차단** (`visibility: hidden` 트릭)

### 카탈로그 / 시리즈 처리
27. **(구) 접두사** → 자동 단종 처리
28. **(뉴) 접두사** → 신버전 인지
29. **2025/2026 라벨 표시 안 함**
30. **온라인 라벨**만 시리즈명 옆에 inline pill로 표시
31. **NEW 버전 라벨 표시 안 함**
32. **R 페어 자동 감지 안 함** (볼케R/볼케S처럼 R이 등급 구분일 수 있음)
33. **카드 우상단 ✕ 버튼**으로 부속품 카드 숨김 (헤이븐 매트리스 전용 ACC, 모션전용 프로텍터 등)
34. **세트마스터 ERP 데이터** 활용 (4107행, 시리즈별 단품코드/사이즈/색상 정확)
35. **단품코드 끝 FA = 아크레 패브릭** 모델 (코펜하겐에서 발견)
36. **숫자/날짜만 있는 시리즈명 자동 제외** (26.04, 2023, 1, P 같은 잔여 텍스트)
37. **레마 같이 여러 그룹에 등장하는 시리즈** 양쪽에 카드 표시

### 사이드바

## 🏗️ 시스템 아키텍처

### 데이터 소스 (5개)

```
1. 학생 일지 (Supabase student_notes)
   ├─ 교육일지 (steps step1/2/3)
   └─ 자율학습 일지

2. 일룸 PPT 자료 (기존자료/*.pptx)
   ├─ ★(통합매장교육TF) 특화모듈_브랜드별교육자료취합본_260126v.pptx (586MB, 986장 → 1~584장만 일룸)
   └─ 일룸 영업전문직(LSA) 입문교육_소재, 브랜드.pptx (62MB, 54장)

3. Notion 단종/사양변경 DB (2-1) 단종품 사양변경 확인 프로젝트)
   ├─ 일룸 단종/변경 단품 현황
   └─ 일룸 단종/변경 세트 현황

4. WordPress 제품 가이드 (https://iloomproduct.fursys.com)
   ├─ 카테고리: 리빙룸/다이닝룸/침실·옷장/키즈룸·틴즈룸/워크룸·멀티룸/펫/조명/공통
   ├─ 시리즈마다 5개 탭 (타겟·기획의도·품목 / 규격·소재 / 특징·주의사항·옵션 / 추천연관제품 / 히스토리)
   └─ 색상 칩 / 제품 사진 / 사이즈 도면 이미지

5. 세트마스터 ERP 엑셀 (기존자료/일룸 세트마스터목록 수파베이스 업로드.xlsx)
   └─ 4107행 (세트코드/색상/사이즈/시리즈/판매채널)
```

### 통합 흐름

```
[5개 데이터 소스]
       ↓
[자동 수집 스크립트들]
       ↓
[Supabase DB + 파일 시스템]
       ↓
[Claude가 일지 분석 + HTML 작성]
       ↓
[textbook_chapters 테이블에 저장]
       ↓
[/dashboard/textbook/[series] 검수/편집]
       ↓
[/dashboard/textbook/print → PDF 인쇄]
```

| 페이지 | 역할 |
|------|------|

### 자동화 스크립트 (scripts/textbook/)

| 스크립트 | 역할 |
|------|------|
| `fetch-catalog.mjs` | 일룸 가이드 카테고리 페이지 크롤링 → 시리즈 목록 수집 (235→222개 정제) |
| `fetch-product-guide.mjs` | WordPress 시리즈 5탭 크롤링 → tab1~5.json 저장 |

### DB 스키마 (Supabase)

```sql
-- 1) PPT 자료 시드
textbook_sources (
  id, series_name, source_type, file_name,
  slides JSONB, full_text TEXT, uploaded_at
)

-- 2) 시리즈 분류 캐시
textbook_classifications (
  id, note_id REFERENCES student_notes(id),
  series_name, confidence, classified_at,
  UNIQUE(note_id, series_name)
)

-- 3) 챕터 본문
textbook_chapters (
  id, series_name UNIQUE, category,
  html_content, status ('draft'/'reviewing'/'final'),
  source_note_ids JSONB, source_pptx_ids JSONB,
  generated_at, updated_at
)
```

## 🤖 자동화 워크플로우

### 현재까지 완성된 자동화

```
[1단계: 카탈로그 수집] (10분, 한 번 실행)
  node scripts/textbook/fetch-catalog.mjs 158 160 154 52 150
    → public/iloom-catalog.json (222개 시리즈)

[2단계: 일지 분류] (5초, 새 일지 추가될 때마다)
  메인 페이지 "전체 일지 분류" 버튼
    → textbook_classifications (정규식 매칭)
    → 시리즈별 일지 수 카운트

[3단계: PPT 등록] (한 번만)
  메인 페이지 "기존 PPT 추출" (또는 source-pptx API)
    → textbook_sources (텍스트 추출 후 저장)
    → 통합본은 1~584장만 (사용자 지정)

[4단계: 시리즈별 챕터 생성] (Claude가 직접 — 코펜하겐 1개 완료)
  ① fetch-series-data.mjs <시리즈명>  → PPT + 일지 추출
  ② fetch-notion-discontinued.mjs <시리즈명>  → 단종 정보
  ③ fetch-product-guide.mjs <p_id>  → WordPress 5탭
  ④ download-images.mjs <p_id> <시리즈명>  → 이미지 + 색상 매칭
  ⑤ Claude가 모든 데이터 + 세트마스터 보고 chapter.html 작성
  ⑥ upsert-chapter.mjs <시리즈명> chapter.html  → DB 저장

[5단계: 검수 + 인쇄]
  /dashboard/textbook/[series]  → 검수/편집
  /dashboard/textbook/print?series=...  → Ctrl+P → PDF 저장
```

### 섹션 순서 (반드시 이 순서)

   - 한 문장 정의 (파란 박스)
