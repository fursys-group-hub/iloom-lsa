# 모바일 UI 가이드 — 일룸 LSA 입문교육

> **2026-04-13 작성.** 모바일/태블릿에서 숨기거나 변환한 요소, 그 이유를 정리한 문서.
> 추후 모바일 수정 작업 시 이 문서를 기준으로 판단할 것.

---

## 1. 반응형 기준점 (Breakpoints)

| 기준 | 적용 내용 |
|------|----------|
| **1024px 이하** | 사이드바 → 햄버거 메뉴로 전환. `hide-mobile` 클래스 적용 시작 |
| **768px 이하** | 콘텐츠 패딩 축소 (32px 40px → 16px 12px), 인사말 가운데 정렬 |

> **왜 1024px?** 사이드바(240px)가 있으면 콘텐츠 영역이 좁아져서, 데스크톱이 아닌 이상 사이드바를 숨기는 게 나음.

---

## 2. 모바일에서 숨긴 요소 (`hide-mobile`)

### 원칙: "모바일은 간단하게 보려고 하는 것"
- 웹에서만 의미 있는 관리 기능은 숨김
- 이름 옆 아바타는 모바일에서 공간만 차지 → 숨김
- 상세 수치(점수, 해당 개수)는 모바일에서 불필요 → 숨김

### 숨긴 목록

| 페이지 | 숨긴 요소 | 이유 |
|--------|----------|------|
| **전체** | 32x32 원형 아바타 | 이름만으로 충분, 모바일에서 가로 공간 절약 |
| **홈** | 교육 일정 타임라인 (입문/심화 날짜 바) | D-day 뱃지("22일차 D-9")로 충분 |
| **홈** | 주의 교육생 상세 ("적응 64점 · 3개 해당") | 뱃지(복합 위기형)만으로 경고 충분 |
| **테스트** | Excel 다운로드 / 오늘 시험 동기화 / 새 응답 추가 | 웹 전용 관리 기능, 모바일에서 실행 불가 |
| **테스트** | 점수 테이블: 제출 시간 칼럼, 오답 칼럼 | 순위/이름/점수/정답만으로 핵심 파악 가능 |
| **출결** | Excel 업로드 버튼 | 웹 전용 기능 (파일 업로드) |
| **실습일지** | 보고서 프롬프트 복사 버튼 | 웹에서 Gemini에 붙여넣기할 때만 필요 |
| **실습일지** | 뱃지 라벨 텍스트 ("상담", "견적", "수주") | 숫자만 표시 (7, 3, 1), 색상으로 구분 가능 |

## 3. 통합한 요약 카드

### 원칙: "카드 4~5개 따로 → 1개 카드 안에 숫자만 한 줄"
- 각각 카드로 분리하면 모바일에서 세로 스크롤이 너무 길어짐
- 1개 카드 안에 그리드로 배치하면 한눈에 파악 가능

| 페이지 | 변경 전 | 변경 후 |
|--------|---------|---------|
| **홈 출결** | 교육인원/출석/지각/결석/퇴사 5개 별도 카드 | 1개 카드, 4~5열 한 줄 |
| **교육일지** | 제출현황 카드 + 완벽참여/거의다/시작 3개 카드 + 추가설명 카드 | 1개 카드, 6열 한 줄 (제출/미제출/완벽/거의다/시작/전체) + 하단 미제출 명단 |
| **실습일지** | 제출현황 카드 + 상담/견적/수주/수주금액 4개 카드 | 1개 카드, 5열 한 줄 (제출/상담/견적/수주/수주금액) + 하단 미제출 명단 |
| **심화교육** | 교육생/평가/총평/벤치마킹 4개 카드 | 1개 카드, 4열 한 줄 |

## 4. 레이아웃 변환

| 페이지 | 데스크톱 | 모바일 (< 1024px) |
|--------|---------|-------------------|
| **사이드바** | 고정 240px 왼쪽 | 햄버거 메뉴 (fixed 상단 고정, 스크롤 따라옴) |
| **홈 메인** | 2컬럼 (좌: 주의교육생+차트 / 우: 교육일지+실적) | 1컬럼 세로 스택 |
| **홈 인사말** | 좌측 정렬 + 우측 D-day/기수 | 가운데 정렬 + 아래로 |
| **개별분석** | 3열 학생 카드 | auto-fill minmax(280px) → 모바일 1열 |
| **테스트 점수** | 6칼럼 (순위/이름/시간/점수/정답/오답) | 4칼럼 (순위/이름/점수/정답) |

## 5. 버튼 스타일 (전체)

| 대상 | 변경 | 이유 |
|------|------|------|
| 수정/삭제/취소/비번초기화 | 테두리 제거 (`border: none`) | 보조 액션은 텍스트만으로 충분 (DESIGN_SYSTEM.md §16) |
| 공지/기수관리 버튼 그룹 | `flexWrap: 'wrap'` 추가 | 좁은 화면에서 자연스럽게 아래로 내려옴 |

## 6. 전역 CSS 규칙 (globals.css)

```css
/* 모바일 테이블 */
@media (max-width: 768px) {
  table { width: 100%; table-layout: auto; }
  td, th { white-space: nowrap; }
}

/* 태블릿 이하 (사이드바 숨김 시점) */
@media (max-width: 1023px) {
  .hide-mobile { display: none !important; }
  .score-row { grid-template-columns: 40px 1fr 80px 60px !important; }
  .score-row > :nth-child(3),
  .score-row > :nth-child(6) { display: none !important; }
}

/* 전역 overflow 방지 */
html, body { overflow-x: hidden; }
```

## 7. 남은 과제

- [ ] 테스트 "시험 분석" 탭 — 차트 2열 → 모바일 1열, 히트맵 가로 스크롤 처리
- [ ] 교육효과(analytics) — 2열 그리드 → 모바일 1열
- [ ] 심화교육(overview) — 6열/4열 그리드 → 모바일 적응
- [ ] manager 총평(final) — 점수 스케일 2열 → 모바일 1열
- [ ] 학생 마이페이지(my/page) — 시험 상세 2열 → 모바일 1열

## 8. 판단 기준 (새 요소 추가 시)

모바일에서 숨길지 판단할 때 이 질문을 하세요:

1. **이 기능을 모바일에서 실행할 수 있는가?** (파일 업로드, 프롬프트 복사 → 불가)
2. **이 정보 없이도 핵심을 파악할 수 있는가?** (아바타, 상세 점수 → 가능)
3. **이 요소가 화면의 30% 이상을 차지하는가?** (5개 카드 → 통합)
4. **같은 정보가 다른 형태로 이미 보이는가?** (타임라인 vs D-day 뱃지 → 중복)
