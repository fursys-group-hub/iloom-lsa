## 0. 골든 스탠다드 (이 품질로 따라가기)

- **로이**: `scripts/textbook/output/series-로이/chapter-claude.html` (318줄, 32~36KB)
- **코펜하겐**: `scripts/textbook/output/series-코펜하겐/` (먼저 만들어진 첫 사례)

새 시리즈를 작성할 때는 **반드시 먼저** 이 두 파일을 열어 구조·톤·이미지 배치 방식을 확인할 것.

### 8섹션 구조 (이 순서 그대로)
```
📋 기본 정보              — 타겟/컨셉/구성
🚨 매장 영업 필수안내       — 단종/후속 모델/주의사항 (일지 100건+에서 패턴 추출)
⭐ 제품 특장점            — 가이드 텍스트를 학습 톤으로 풀어쓰기 + 이미지
💡 매장 실전 팁           — 견적 → 안전등급 → 실측 인과 흐름
🎯 학생들이 헷갈리는 포인트 — 시험·일지·Q&A 빈출 패턴
🎯 vs 다른 시리즈         — 포지셔닝 (학생방이면 학생방끼리 비교)
🎨 색상 가이드            — 베이스+액세서리 조합 + 칩 이미지 (가로 정렬)
🧵 소재 가이드 (소파만)    — 패브릭/가죽 코드별 (소파 시리즈에만 해당)
─── 사양 정보 (Reference) ───  ← 여기서부터 자동 생성
🪑 운영 라인업 (단품 코드 + 사이즈)
📐 치수 / 호환성 표
```

## 1. 자동화 파이프라인 (스크립트로 한 번에)

다음 5단계를 순서대로 실행하면 `chapter-claude.html` 작성 직전 상태까지 자동으로 도달함.

### 1단계: 데이터 수집

```bash
node scripts/textbook/collect-series-data.mjs <시리즈명>
# 예: node scripts/textbook/collect-series-data.mjs 뉴트
```

수집 소스 4개 → `output/series-<시리즈명>/all-data.json`:
1. **카탈로그**: `public/iloom-catalog.json` — page_id, category, pumok, gubun, 단종/온라인 여부
2. **textbook_product_guide**: 5탭 본문/표/이미지 + sub_pages (일룸 가이드 사이트 크롤링 결과)
3. **textbook_set_master**: 시리즈명 매칭 단품들 (단품코드/색상/사이즈/채널)
4. **textbook_classifications + student_notes**: 분류된 일지 본문 (일지 100건+에서 패턴 추출용)

> **시리즈 별칭 처리**: `lib/series-aliases.ts`를 참고해서 분류 별칭도 함께 검색됨.
> 예: 뉴트 → ['뉴트 홈오피스']도 같이 검색

### 2단계: sub-page 상세 크롤링 (필요시)

서브 페이지(예: 로이 측판형 책상, 로이 다리형 책상…)가 많은 시리즈는:

```bash
node scripts/textbook/fetch-sub-page-details.mjs <시리즈명>
```

→ `output/series-<시리즈명>/sub-pages-detail.json` 생성. 학생들이 헷갈려하는 차이점(예: 측판형 vs 다리형) 분석에 사용.

### 3단계: sceleton 자동 생성

```bash
node scripts/textbook/generate-chapter.mjs <시리즈명>
```

→ `output/series-<시리즈명>/chapter-auto.html` 생성. 8섹션 구조 + 자동 채울 수 있는 부분(타겟, 구성, 색상 맵 등)은 채워짐.

### 4단계: 이미지 다운로드 + Storage 업로드 ⚠️ 잊지 말 것 ⚠️

**가장 자주 빼먹는 단계.** 수지님이 매번 "이미지가 왜 없냐"고 묻는 이유. 반드시 실행할 것.

각 page_id마다:

```bash
node scripts/textbook/download-images.mjs <page_id> <시리즈명>
# 예: node scripts/textbook/download-images.mjs 2381 로이
```

동작:
- 일룸 가이드 사이트(WordPress) 로그인 → 5탭 순회 → 모든 이미지 다운로드
- Supabase Storage `textbook-images/p<pid>/<파일명>`로 업로드
- 메타 저장: `output/series-<시리즈명>/images-meta.json`
  - 각 이미지마다 `public_url`, `parent_text`(직전 텍스트), `tab`(어느 탭) 정보 포함
  - 이 메타가 **chapter-claude.html에 어떤 이미지를 어디에 넣을지 결정하는 핵심 자료**

> **여러 page_id가 있는 시리즈** (예: 로이는 p=2381 메인 + sub들):
> 메인 page_id만 download-images 돌리면 충분. sub들의 이미지가 메인 페이지 5탭에 다 들어 있음.

### 5단계: 색상/소재 라이브러리 보강 (이미 있으면 스킵)

```bash
# 색상 칩 (전 시리즈 공유) — colors/SP.jpg, OS.jpg, GU.jpg, GY.jpg, LU.png, DN.png
node scripts/textbook/setup-color-library.mjs list  # 등록된 칩 목록
node scripts/textbook/setup-color-library.mjs add SP p15137/04_243x243_tab1.jpg  # 새 칩 추가

# 소파 소재 (이미 60개 등록됨, 추가 작업 불필요)
# lib/material-images.ts 참조
```

- **결과물 위치**: `lib/color-chips.ts`, `lib/material-images.ts` (모든 시리즈가 공유)
- **이미지 정책**: 갤러리 모든 이미지는 **aspect-ratio:1/1 + object-fit:cover**로 통일 (수지님이 "사이즈 다 동일하게" 강조)

### 6단계: chapter-claude.html 작성 (Claude 1:1 직접)

이게 핵심 작업. 아래 §2 참고.

### 7단계: auto-spec 머지 + DB upsert

```bash
node scripts/textbook/merge-chapter-with-spec.mjs <시리즈명>
```

동작:
- `chapter-claude.html` + 자동 사양 섹션(라인업/치수) 머지 → `chapter-final.html`
- `textbook_chapters` 테이블에 upsert (status: `reviewing`)
- 메인 페이지 카드가 자동으로 파란 "검수중 →" 버튼으로 변경됨

> **수동 upsert가 필요한 경우**: `node scripts/textbook/upsert-chapter.mjs <시리즈명> <html파일경로>`

## 2. chapter-claude.html 작성 가이드 (Claude가 직접 쓰는 부분)

### 톤 원칙 (수지님이 반복 피드백한 것)

| 원칙 | 예시 |
|------|------|
| **"왜"를 설명** | "가로 공간 효율" ❌ → "다리 대신 측면 패널로 받침 → 양 옆 막혀있음 → 옆에 다른 가구 붙여놓고 싶은 좁은 방" ✅ |
| **코드 외우기 X, 호환성 O** | 단품코드 표 ❌ → "어떤 책상장이 어떤 책상에 맞나" 호환성 표 ✅ |
| **인과 흐름** | "안전등급 / 실측 / 견적" 별개 섹션 ❌ → "안전등급 결정 → 그래서 실측 시 이걸 확인" 한 흐름 ✅ |
| **수치 차이는 이유 설명** | "1400 vs 1447" ❌ → "책상장은 책상 위에 결합되며 책상보다 약간 더 넓음 (1400 책상 → 1447 책상장)" ✅ |
| **색상은 베이스+액세서리** | "로이뮤트 투톤" 같은 특수 라벨 ❌ → "SPGY = SP 베이스 + GY 액세서리" 일반화 ✅ |
| **불릿 살리기** | 불릿 다 없애지 말 것. CSS도 `list-style: disc !important; display: list-item !important;` |
| **장식 이모지 X** | UI 장식용 이모지 금지. 단 8섹션 헤더의 📋⭐💡🎯🎨 등은 헤더 식별용이므로 OK |
| **NEW 배지 안 붙임** | 명시 요청 없으면 NEW/🆕 안 붙임 |

### 섹션별 작성 방법

#### 📋 기본 정보
- `all-data.json`의 `product_guide.tabs[0]` (메인 탭) 첫 단락에서 타겟·컨셉 발췌
- 일지/메모로 보강 (예: "신혼 ~ 영유아 자녀")
- 이미지: 메인 hero 1~2장 (`images-meta.json`에서 tab=1, 큰 사이즈)

#### 🚨 매장 영업 필수안내
- `all-data.json`의 `notes` 배열에서 패턴 추출:
  - "단종" 키워드
  - "후속 모델" / "변경"
  - "리콜" / "주의" / "고객 컴플레인"
- 영업사원 관점에서 "이거 모르면 큰일 나는" 항목만 골라서 정리
- **회사 정책 톤은 수지님이 검수 단계에서 잡음** — 일단 일지에 있는 사실 그대로 작성

#### ⭐ 제품 특장점
- `product_guide.tabs[1~3]`의 본문 텍스트 활용
- "왜"를 항상 풀어쓰기. 영업사원이 고객에게 그대로 전달할 수 있는 톤
- 이미지: `images-meta.json`에서 해당 특징 옆 이미지 매칭 (`parent_text`로 위치 파악)

#### 💡 매장 실전 팁
- **인과 흐름 한 줄로**: 견적 → 안전등급 결정 → 실측 시 체크 → 시공
- 별개 섹션으로 쪼개지 말 것
- 일지에서 "실수했다" / "조심해라" 패턴 추출

#### 🎯 학생들이 헷갈리는 포인트
- `student_notes`의 step3(헷갈렸던 부분), `student_questions` 활용
- 비교 문제는 `+표기` (예: "뉴트+로이")
- 시험 빈출 패턴은 P5 questions 666개에서 series_name 일치 항목 확인

#### 🎯 vs 다른 시리즈
- 같은 카테고리 시리즈와 비교 (학생방이면 학생방끼리)
- 가격대 / 타겟 연령 / 핵심 차별점 3축 정도로 정리

#### 🎨 색상 가이드
- **가로 정렬** (수지님: "왼쪽에서 오른쪽으로 가게")
- 칩은 `lib/color-chips.ts`의 Storage URL 사용
- 베이스+액세서리 조합 명시 (예: SPGY = SP 베이스 + GY 액세서리)

#### 🧵 소재 가이드 (소파 시리즈만)
- `lib/material-images.ts`의 60개 코드 활용
- 시리즈별로 사용되는 코드만 필터 (`product_guide`에 언급된 것)

### 이미지 사용 규칙

```html
<!-- 갤러리 그리드 (수지님이 "사이즈 다 동일하게" 강조) -->
<div class="img-gallery" style="display:grid;grid-template-columns:repeat(auto-fill,minmax(180px,1fr));gap:8px;">
  <img src="{Storage public_url}" style="aspect-ratio:1/1;object-fit:cover;width:100%;border-radius:8px;cursor:zoom-in;">
  ...
</div>
```

- **반드시 Storage URL 사용** (`https://...supabase.co/storage/v1/object/public/textbook-images/...`)
- `/textbook-images/...` 같은 상대경로 쓰지 말 것 → `rewrite-chapter-image-paths.mjs`로 보정해야 함
- 이미지 클릭 시 lightbox 자동 (검수 페이지에 이미 구현됨)

## 3. "우선순위 시리즈 자동으로 진행해" 한 마디로 실행하는 흐름

수지님이 이 한 마디만 하면 다음 5개를 끝까지 진행:

**우선순위**: 뉴트 → 링키플러스 → 에디 → 에디키즈 → 팅클팝

### 시리즈 한 개당 실행 순서 (Claude가 알아서)

```bash
# (시리즈명 = 뉴트 예시)
SERIES="뉴트"

# 1) 데이터 수집
node scripts/textbook/collect-series-data.mjs "$SERIES"

# 2) page_id 확인 (all-data.json의 cards[].page_id)
#    여러 개면 메인 page_id 1개 골라서 다음 단계

# 3) sub-page 상세 (sub 많으면)
node scripts/textbook/fetch-sub-page-details.mjs "$SERIES"

# 4) sceleton 생성
node scripts/textbook/generate-chapter.mjs "$SERIES"

# 5) ⚠️ 이미지 다운로드 (메인 page_id) — 절대 빼먹지 말 것
node scripts/textbook/download-images.mjs <page_id> "$SERIES"

# 6) Claude가 chapter-claude.html 직접 작성 (§2 가이드 따라)
#    output/series-뉴트/chapter-claude.html

# 7) 머지 + DB upsert (status: reviewing)
node scripts/textbook/merge-chapter-with-spec.mjs "$SERIES"
```

이 7단계를 5개 시리즈 반복.

### 매번 결과 확인 체크리스트

작성한 chapter-claude.html을 머지하기 전에 self-check:
- [ ] 이미지가 들어가있는가? (Storage URL `https://...supabase.co/...`)
- [ ] 갤러리 이미지 사이즈 통일? (`aspect-ratio:1/1`, `object-fit:cover`)
- [ ] 8섹션 모두 채워졌는가?
- [ ] "왜"가 설명되어 있는가? (가로 공간 효율 같은 모호한 표현 없는가)
- [ ] 코드 표 대신 호환성 표를 썼는가?
- [ ] 견적/안전등급/실측이 한 흐름으로 묶여있는가?
- [ ] 불릿이 살아있는가? (`list-style: disc !important` 적용)
- [ ] 색상이 가로 정렬인가?
- [ ] 베이스+액세서리 조합 설명이 있는가?

## 4. 사이트 구조의 미묘한 차이 — 알려진 케이스

| 케이스 | 처리 |
|--------|------|
| 뉴트 홈오피스가 뉴트 sub로 묶임 | `lib/series-aliases.ts`의 `'뉴트': ['뉴트 홈오피스']` 별칭 매핑 |
| 케플러클래식 / 케플러 클래식 띄어쓰기 | `'케플러클래식': ['케플러 클래서']` |
| 글렌 라이브러리 / 글렌 | `'글렌 라이브러리': ['글렌']` |
| 1글자 시리즈명 (닛, 로) | series-map.ts에서 단어 경계 정규식 매칭 |
| 같은 시리즈명 다중 카드 (예: 레마 식탁 + 레마 거실수납장) | `subsByNameCount` + PUMOK_KEYWORDS로 카드별 sub 분리 |
| 부분 매칭 오염 (예: 콜로이 → 로이로 매칭) | 단어 경계 정규식으로 차단 |
| 데일리 라이트 = 온라인 전용 | `SERIES_OVERRIDES`의 `force_online: true` |
| HCH0020W 단종이지만 HCH0020WN/G 활성 | 별칭 유지: `'버튼': ['버튼스위블']` |

새 패턴 발견 시:
- 별칭 갭이면 → `lib/series-aliases.ts` 추가
- 카탈로그 표기 갭이면 → `scripts/textbook/fetch-catalog.mjs`의 `SERIES_OVERRIDES` 추가
- 분류 정규식 갭이면 → `lib/series-map.ts` 갱신 후 `/api/textbook/classify --force` 재실행

## 5. DB 스키마 메모

| 테이블 | 역할 |
|--------|------|
| `textbook_chapters` | 챕터 본문 HTML + status (draft/reviewing/final) — **메인 결과물** |
| `textbook_product_guide` | 일룸 가이드 사이트 5탭 크롤링 결과 |
| `textbook_set_master` | 4,107행 단품 마스터 (단품코드/색상/사이즈/채널) |
| `textbook_classifications` | 일지 ↔ 시리즈 자동 분류 결과 |

### chapter status 의미
- `draft`: 자동 sceleton만. 카드 회색 "초안 작성 중"
- `reviewing`: Claude가 1:1 작성 완료. 카드 파란 "검수중 →"
- `final`: 수지님 검수 완료. 카드 초록 "완료 (편집)"

## 7. 절대 잊지 말 것 (수지님이 반복 지적한 것)

1. **이미지 빼먹지 말 것** — `download-images.mjs` 안 돌리면 챕터에 이미지 0장이 됨. 매번 새 시리즈마다 실행
2. **이미지 사이즈 통일** — `aspect-ratio:1/1 + object-fit:cover`. 들쭉날쭉 금지
3. **색상은 가로 정렬** — 위→아래 순서가 아닌 왼쪽→오른쪽
4. **코드 외우기 강요 X** — 학생들은 코드 외울 때가 아님. 호환성/치수/구조 위주
5. **"왜"를 설명** — 모든 수치/특징은 이유 명시
6. **불릿 살리기** — `list-style: disc !important` 빠뜨리지 말 것
7. **NEW/🆕 배지 안 붙임** — 명시 요청 없으면 기본 안 붙임
8. **장식 이모지 X** — 8섹션 헤더 외엔 본문에 이모지 안 씀
