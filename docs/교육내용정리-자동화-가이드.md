## 0. 골든 스탠다드 (이 품질로 따라가기)

- **로이**: `scripts/textbook/output/series-로이/chapter-claude.html` (~50KB, 영업 시나리오 기반 8섹션)
- **코펜하겐**: `scripts/textbook/output/series-코펜하겐/` (먼저 만들어진 첫 사례)

새 시리즈를 작성할 때는 **반드시 먼저** 이 두 파일을 열어 구조·톤·이미지 배치 방식을 확인할 것.

### ⚡ 작성 전 필수 단계 — 기존 PPTX 비교

**`기존자료/★(통합매장교육TF) 특화모듈_브랜드별교육자료취합본_260126v.pptx`** 에는 PART별로 시리즈별 슬라이드가 정리되어있음 (986슬라이드).

새 시리즈 작성 전 반드시:
1. PPTX에서 해당 시리즈 슬라이드 텍스트 추출 (`adm-zip` + `<a:t>` 태그 매칭)
2. 추출 텍스트를 일룸 가이드(`product_guide.tabs`)와 비교
3. **PPTX에만 있는 영업 디테일 골라서 챕터에 통합** — 빠지면 안 되는 항목:
   - 모터 브랜드 / AS 기간 / 높이 조절 범위
   - 액세서리 우측/좌측 고정, 기본 포함 여부
   - 멀티탭 구수 + 위치 (책상별)
   - 가방걸이 운영 방식 (책상 종류별 다름)
   - 책장·도어 구성 규칙 (5단+3도어 같은)
   - 시리즈 비교 표 (vs 모노/뮤트/미니 등 별도 변형)

### 8섹션 구조 (영업 시나리오 기반 — 2026-04-28 확정)

영업사원이 매장에서 고객을 응대하는 순서대로:

```
1. 📋 한눈에 보기
   ├─ 한 문장 정의 (이 시리즈가 뭔지)
   ├─ 모델명 / 카테고리 / 타겟 고객 / 컨셉 / 구성
   ├─ 색상 6종 미리보기
   ├─ 차별 포인트 / 판매채널 / 관련 단품 수
   └─ 가이드 사이트 링크

2. 🪑 라인업 — 책상 4종 + 수납 + 옷장 (제품 구조 중심)
   ├─ 책상 4종 비교표 (사진 + 특징 + 멀티탭 + 추천 고객)
   ├─ 모션데스크 핵심 스펙 (높이 조절, 모터, AS, 사이즈 유의)
   ├─ 책상장·적층장 표 (단수/폭/액세서리/멀티탭/배선홀)
   ├─ 책장 표 (도어 구조 / 도어 구성 규칙 / 힌지 브랜드)
   ├─ 서랍장·거울장·전신거울 갤러리
   └─ 통합 옷장 좁은장 (간략, 옷장은 별 시리즈)

3. 🔌 매장 영업 빈출 Q&A — AS·멀티탭·가방걸이·배치
   ├─ ① 모션데스크 AS 정책 (모터 5년/컨박 3년/소모품 1년)
   ├─ ② 멀티탭 위치 (책상별 어디에)
   ├─ ③ 가방걸이 운영 (측판형 X, 다리형/모션 O, 링플 호환)
   ├─ ④ 책상 배치 3가지 패턴 (일자/ㄱ자/독립)
   └─ ⑤ 사이즈 유의 (1200=실제 1160 등)

4. 💡 견적·시공 실전 — 안전등급부터 실측까지
   ├─ 1️⃣ 안전등급 결정 (권장↔필수, 이유 설명)
   ├─ 2️⃣ 도어 부착 규칙 (책상장 1200공통/1400구분, 책장, 익스텐션 호환)
   ├─ 3️⃣ 익스텐션 시공 유의 (회전불가/높이조절불가)
   └─ 4️⃣ 안전등급 → 실측 체크리스트

5. 🌈 색상·조합 가이드
   ├─ 6색 칩 (가로 정렬, aspect-ratio:1/1)
   ├─ 베이스+액세서리 조합 매트릭스
   ├─ 실제 조합 적용 사진 (SPGY, OSGY 등)
   └─ 시리즈 별도 컬러 라인 (로이뮤트 = 중·고등 타겟)

6. 🎯 시험·매장 빈출 헷갈리는 포인트
   ├─ 책상별 사이즈 (폭 × 깊이)
   ├─ 책상장 호환성 표
   ├─ 책상장 vs 적층장 (1400 vs 1447)
   └─ 책장 호환성 (도어 부착 규칙)

7. 🎯 vs 다른 학생방 시리즈 (포지셔닝)
   ├─ 로이 vs 뉴트 vs 링키플러스 종합 비교표
   ├─ 로이 vs 로이모노 상세 비교 (깊이/색상/가격/구성/액세서리)
   └─ 추천 고객 페르소나

8. 🚨 단종·주의·매장 영업 필수 안내
   견적·시공 전 체크리스트 (5~8개 핵심 항목)

─── 사양 정보 (Reference) ───  ← 여기서부터 자동 생성
🪑 운영 라인업 (단품 코드 + 사이즈)
📐 치수 / 호환성 표
```

### 왜 이 구조인가 (설계 원칙)

- **영업 흐름 기반**: 매장 응대 순서(개요→설명→Q&A→견적→컬러→시험→비교→경고)대로
- **AS·멀티탭·가방걸이·배치를 한 곳에**: 영업 빈출 질문이 흩어지지 않음 (3번 섹션)
- **vs 시리즈 통합**: 다른 학생방 시리즈 비교 + 로이 vs 로이모노 상세 표를 한 섹션에 (7번)
- **필수 안내가 마지막**: 견적 전 체크리스트로 활용 가능 (8번)
- **이미지 통일**: 모든 갤러리 이미지 `aspect-ratio:1/1` + `object-fit:cover`

## 1. 자동화 파이프라인 (스크립트로 한 번에)

다음 5단계를 순서대로 실행하면 `chapter-claude.html` 작성 직전 상태까지 자동으로 도달함.

### 1단계: 데이터 수집

```bash
node scripts/textbook/collect-series-data.mjs <시리즈명>
# 예: node scripts/textbook/collect-series-data.mjs 뉴트
```

수집 소스 4개 → `output/series-<시리즈명>/all-data.json`:
1. **카탈로그**: `public/iloom-catalog.json` — page_id, category, pumok, gubun, 단종/온라인 여부
2. **textbook_product_guide**: 5탭 본문/표/이미지 + sub_pages (일룸 가이드 사이트 크롤링 결과)
3. **textbook_set_master**: 시리즈명 매칭 단품들 (단품코드/색상/사이즈/채널)
4. **textbook_classifications + student_notes**: 분류된 일지 본문 (일지 100건+에서 패턴 추출용)

> **시리즈 별칭 처리**: `lib/series-aliases.ts`를 참고해서 분류 별칭도 함께 검색됨.
> 예: 뉴트 → ['뉴트 홈오피스']도 같이 검색

### 2단계: sub-page 상세 크롤링 (필요시)

서브 페이지(예: 로이 측판형 책상, 로이 다리형 책상…)가 많은 시리즈는:

```bash
node scripts/textbook/fetch-sub-page-details.mjs <시리즈명>
```

→ `output/series-<시리즈명>/sub-pages-detail.json` 생성. 학생들이 헷갈려하는 차이점(예: 측판형 vs 다리형) 분석에 사용.

### 3단계: sceleton 자동 생성

```bash
node scripts/textbook/generate-chapter.mjs <시리즈명>
```

→ `output/series-<시리즈명>/chapter-auto.html` 생성. 8섹션 구조 + 자동 채울 수 있는 부분(타겟, 구성, 색상 맵 등)은 채워짐.

### 4단계: 이미지 다운로드 + Storage 업로드 ⚠️ 잊지 말 것 ⚠️

**가장 자주 빼먹는 단계.** 수지님이 매번 "이미지가 왜 없냐"고 묻는 이유. 반드시 실행할 것.

각 page_id마다:

```bash
node scripts/textbook/download-images.mjs <page_id> <시리즈명>
# 예: node scripts/textbook/download-images.mjs 2381 로이
```

동작:
- 일룸 가이드 사이트(WordPress) 로그인 → 5탭 순회 → 모든 이미지 다운로드
- Supabase Storage `textbook-images/p<pid>/<파일명>`로 업로드
- 메타 저장: `output/series-<시리즈명>/images-meta.json`
  - 각 이미지마다 `public_url`, `parent_text`(직전 텍스트), `tab`(어느 탭) 정보 포함
  - 이 메타가 **chapter-claude.html에 어떤 이미지를 어디에 넣을지 결정하는 핵심 자료**

> **여러 page_id가 있는 시리즈** (예: 로이는 p=2381 메인 + sub들):
> 메인 page_id만 download-images 돌리면 충분. sub들의 이미지가 메인 페이지 5탭에 다 들어 있음.

### 5단계: 색상/소재 라이브러리 보강 (이미 있으면 스킵)

```bash
# 색상 칩 (전 시리즈 공유) — colors/SP.jpg, OS.jpg, GU.jpg, GY.jpg, LU.png, DN.png
node scripts/textbook/setup-color-library.mjs list  # 등록된 칩 목록
node scripts/textbook/setup-color-library.mjs add SP p15137/04_243x243_tab1.jpg  # 새 칩 추가

# 소파 소재 (이미 60개 등록됨, 추가 작업 불필요)
# lib/material-images.ts 참조
```

- **결과물 위치**: `lib/color-chips.ts`, `lib/material-images.ts` (모든 시리즈가 공유)
- **이미지 정책**: 갤러리 모든 이미지는 **aspect-ratio:1/1 + object-fit:cover**로 통일 (수지님이 "사이즈 다 동일하게" 강조)

### 6단계: chapter-claude.html 작성 (Claude 1:1 직접)

이게 핵심 작업. 아래 §2 참고.

### 7단계: auto-spec 머지 + DB upsert

```bash
node scripts/textbook/merge-chapter-with-spec.mjs <시리즈명>
```

동작:
- `chapter-claude.html` + 자동 사양 섹션(라인업/치수) 머지 → `chapter-final.html`
- `textbook_chapters` 테이블에 upsert (status: `reviewing`)
- 메인 페이지 카드가 자동으로 파란 "검수중 →" 버튼으로 변경됨

> **수동 upsert가 필요한 경우**: `node scripts/textbook/upsert-chapter.mjs <시리즈명> <html파일경로>`

### 섹션별 작성 방법

#### 1. 📋 한눈에 보기
- 한 문장 정의 먼저 (영업사원이 외워서 말할 수 있는 한 줄)
- 메타 표: 모델명 / 카테고리 / 타겟 / 컨셉 / 구성 / 색상 / 차별 포인트 / 판매채널 / 관련 단품 수 / 가이드 사이트
- `all-data.json`의 `product_guide.tabs[0]` 첫 단락 + `cards[].pumok/gubun` 활용
- 별도 변형 시리즈 언급 (예: 로이의 로이모노)

#### 2. 🪑 라인업
- 책상/수납/옷장/침대 등 제품군별로 정리
- **각 제품군마다 표 + 이미지 갤러리 한 세트**
- 모션데스크처럼 핵심 스펙이 많으면 **별도 표** (높이/모터/AS/사이즈 유의)
- PPTX의 `종류 및 사이즈` / `특징` / `구성 옵션` 섹션이 핵심 소스
- 수지님 강조: **"왜"를 풀어쓰기** — "측판형은 다리 대신 측면 패널 받침 → 양 옆 막힘 → 좁은 방에 적합"

#### 3. 🔌 매장 영업 빈출 Q&A
- **AS·멀티탭·가방걸이·배치를 모두 여기에 모음**
- AS는 표로 (부품별 기간 + 시공일 기준 명시)
- 멀티탭은 책상 종류별로 (위치 + 구수 + 비고)
- 가방걸이는 책상 종류별로 (운영 방식 다름 + 호환 시리즈 명시)
- 책상 배치 3가지 패턴 (일자/ㄱ자/독립) — 매장 도면 그릴 때 사용
- 사이즈 유의 한 줄 (1200=실제 1160)

#### 4. 💡 견적·시공 실전

### 표 셀 줄바꿈 규칙

표 안의 한 셀에 **본 내용 + 부가 설명(괄호)** 이 같이 들어갈 때는 줄바꿈으로 분리:

```html
<!-- ❌ 한 줄에 다 붙어있음 -->
<td>책상보다 약간 더 넓음 (1400 책상 → 1447 책상장으로 양옆 살짝 돌출)</td>

<!-- ✅ 본 내용 / 부가 설명 분리 -->
<td>책상보다 약간 더 넓음<br/>(1400 책상 → 1447 책상장으로 양옆 살짝 돌출)</td>
```

→ 표 가로폭이 길어지지 않고 시각적으로도 핵심·부가 분리됨.

**박스 클래스**:
- `.alert-note` (빨간) — 안전 규정·견적 누락·필수 안내
- `.tip-note` (노란) — 영업 멘트·매장 응대·시연 멘트
- `.info-note` (파란) — 개념 설명·"❓" 질문 박스

### 표 컬럼 너비 명시 규칙 (가독성 핵심)

  </thead>
</table>

<!-- 또는 thead th에 직접 width -->
<thead><tr><th style="width:18%;">옷장 종류</th><th style="width:25%;">사이즈 (W × D × H)</th><th>특징</th></tr></thead>

### 작성 순서 매트릭스 (8단계로 나눠서 처음부터 완성도)

작성하는 순서가 중요. 한 번에 다 쓰지 말고 단계별로 차근차근:

| 단계 | 작업 | 적용 패턴 |
|------|------|----------|
| **1️⃣ 골격 잡기** | 8섹션 헤더 + 한 문장 정의 + 메타 표 골격 | 한눈에 보기 미니 색상 칩 표(가로 정렬)·차별 포인트 5~6개 |
| **2️⃣ 라인업 비교표** | 책상·옷장·침대 등 카테고리별 비교표 | **표 thead width 명시 + 사이즈 셀 nowrap + 특징 컬럼 불릿 분리** |
| **3️⃣ 항목별 이미지+표** | 각 sub-section마다 [이미지 → 표] 패턴 (수납장 5종 같은 경우) | 큰 갤러리 X · 항목 바로 위에 이미지 1장 |
| **4️⃣ 디테일 갤러리** | 사용 시퀀스(레이어드 패널처럼 ①~④) / 스펙별 컷 (사선엣지·HPM·배선캡 등) | figcaption: 굵은 제목 + small 부가 |
| **5️⃣ "왜?" 풀어쓰기** | 모든 수치·구조에 이유 + 매장 시연 멘트 (3단 구조) | D740·1400 vs 1447·길이 차이 등 |
| **6️⃣ 비교 박스 (동일 카테고리 다른 이름)** | "❓ 다른 거예요?" info-note + 비교표 | 책상장 vs 적층장·다리형 스크린 vs 모션 스크린 등 |
| **7️⃣ details 토글 (호환 불가 이유)** | 핵심 강조 + "왜? 클릭하세요" 펼침 | CSS 룰 사용 / 중복 정보 통합 |
| **8️⃣ 한눈에 보기 매트릭스** | 헷갈리는 포인트 섹션 시작에 카테고리 그룹핑 매트릭스 | 시험·견적 암기용 |

## 4. 사이트 구조의 미묘한 차이 — 알려진 케이스

| 케이스 | 처리 |
|--------|------|
| 뉴트 홈오피스가 뉴트 sub로 묶임 | `lib/series-aliases.ts`의 `'뉴트': ['뉴트 홈오피스']` 별칭 매핑 |
| 케플러클래식 / 케플러 클래식 띄어쓰기 | `'케플러클래식': ['케플러 클래서']` |
| 글렌 라이브러리 / 글렌 | `'글렌 라이브러리': ['글렌']` |
| 1글자 시리즈명 (닛, 로) | series-map.ts에서 단어 경계 정규식 매칭 |
| 같은 시리즈명 다중 카드 (예: 레마 식탁 + 레마 거실수납장) | `subsByNameCount` + PUMOK_KEYWORDS로 카드별 sub 분리 |
| 부분 매칭 오염 (예: 콜로이 → 로이로 매칭) | 단어 경계 정규식으로 차단 |
| 데일리 라이트 = 온라인 전용 | `SERIES_OVERRIDES`의 `force_online: true` |
| HCH0020W 단종이지만 HCH0020WN/G 활성 | 별칭 유지: `'버튼': ['버튼스위블']` |

새 패턴 발견 시:
- 별칭 갭이면 → `lib/series-aliases.ts` 추가
- 카탈로그 표기 갭이면 → `scripts/textbook/fetch-catalog.mjs`의 `SERIES_OVERRIDES` 추가
- 분류 정규식 갭이면 → `lib/series-map.ts` 갱신 후 `/api/textbook/classify --force` 재실행

## 7. 절대 잊지 말 것 (수지님이 반복 지적한 것)

1. **이미지 빼먹지 말 것** — `download-images.mjs` 안 돌리면 챕터에 이미지 0장이 됨. 매번 새 시리즈마다 실행
2. **이미지 사이즈 통일** — `aspect-ratio:1/1 + object-fit:cover`. 들쭉날쭉 금지
3. **색상은 가로 정렬** — 위→아래 순서가 아닌 왼쪽→오른쪽
4. **코드 외우기 강요 X** — 학생들은 코드 외울 때가 아님. 호환성/치수/구조 위주
5. **"왜"를 설명** — 모든 수치/특징은 이유 명시
6. **불릿 살리기** — `list-style: disc !important` 빠뜨리지 말 것
7. **NEW/🆕 배지 안 붙임** — 명시 요청 없으면 기본 안 붙임
8. **장식 이모지 X** — 8섹션 헤더 외엔 본문에 이모지 안 씀
