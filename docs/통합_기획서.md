# 일룸 신입사원 교육 관리 시스템 — 통합 기획서

> **작성일**: 2026-04-26  
> **작성자**: 김수지 (교육 담당)  
> **시스템 주소**: https://iloom-lsa-production.up.railway.app  
> **구축 기간**: 2026.03.26 ~ 현재 (약 1개월)

---

## 1. 이 시스템을 왜 만들었는가

### 내가 겪은 문제들

나는 일룸 신입사원(LSA) 입문교육을 담당하고 있다. 교육생은 한 기수에 10~20명 정도인데, 교육 기간 동안 매일 이런 일을 반복했다:

- **시험 채점**: 구글 폼으로 시험 보고, 결과를 엑셀로 옮겨서 차시별로 정리. 14차시를 하니까 매번 30분~1시간씩 걸렸다.
- **출결 확인**: 타임인아웃(출결 앱)에 들어가서 엑셀 다운받고, 누가 지각했는지 확인. 매일.
- **교육일지 확인**: 노션에 교육생마다 개별 페이지가 있어서, 하나하나 들어가서 확인. 누가 열심히 썼는지, 누가 안 썼는지 한눈에 보기 어려웠다.
- **피드백**: 카톡으로 개별 메시지. 누구에게 뭘 말했는지 기록이 안 남는다.
- **리포트 작성**: 교육이 끝나고 매장에 교육생을 보낼 때, 매장 관리자에게 "이 사람은 이런 사람이에요"를 전달해야 하는데, 워드로 한 명씩 쓰려니까 엄두가 안 났다.

가장 큰 문제는 **"교육이 효과 있었나요?"**에 답할 수 없다는 것이었다. 느낌상 잘한 것 같은데, 데이터가 없으니 증명할 방법이 없었다. 기수가 쌓이면 1기랑 2기를 비교할 수도 없고.

### 그래서 만들기로 했다

코딩을 할 줄 모르지만, AI 도구(Claude Code)를 써서 "이런 기능이 필요해"라고 설명하면 AI가 코드를 만들어준다. 외주를 맡기면 몇 천만 원이 드는데, 내가 교육 현장을 가장 잘 아니까 직접 만들면 현장에 딱 맞는 도구가 나올 거라고 생각했다. 

결과적으로 약 1개월 만에, 교육 운영에 필요한 거의 모든 기능을 갖춘 시스템을 만들었다.

## 2. 만드는 과정 — 뭘 했고, 어떤 어려움이 있었는가

### 구축 타임라인

| 날짜 | 한 일 | 삽질/어려움 |
|------|-------|-----------|
| **3/26** | 기획서 작성, 프로젝트 시작 | 뭘 먼저 해야 할지 감이 안 잡힘 |
| **3/27** | 기본 뼈대 완성 — 로그인, 대시보드, 교육생 관리, 출결, 시험 동기화 | 데이터베이스 설계가 어려웠음. 날짜 형식이 안 맞아서 한참 헤맸다 |
| **3/28** | 교육생 전용 페이지 + AI 분석 기능 | 교육생이 볼 수 있는 범위 vs 교육자만 볼 수 있는 범위 구분이 필요했다 |
| **3/30~31** | 교육일지 시스템 전면 구축 + 디자인 개선 | 노션에 있던 교육일지 82개를 새 시스템으로 옮겨야 했다. 자동 변환으로 해결 (수작업이었으면 3일 걸릴 것을 30분에) |
| **4/01** | 소통 기능 — 공지사항 + 교육일지 코멘트 + 모바일 대응 | 모바일에서 레이아웃이 깨지는 문제. 화면 크기별로 다 조정해야 했다 |
| **4/02** | 운영 도구 완성 — 실습일지 + 질문관리 + 교육자 메모 | 기능이 많아지면서 홈 화면을 어떻게 구성할지 고민. 2컬럼 레이아웃으로 해결 |
| **4/05** | 구글 폼 시험 채점 시스템 분석 | 기존 채점 방식이 7가지(OX, 객관식, 주관식 등)나 되어서 복잡. 분석만 하루 걸렸고, 결국 별도 프로젝트로 분리하기로 결정 |
| **4/06~07** | AI 코칭 리포트 시스템 | 14명분 종합 분석 리포트를 AI로 자동 생성. 교육자 메모를 생활기록부 톤으로 리라이팅하는 게 핵심이었다 |
| **4/08** | 자동화 완성 — 출결 자동 수집 + 시험 자동 동기화 | 자동 로봇(Playwright)이 타임인아웃에서 출결을 자동으로 가져오게 만들었다. 하루 4번 자동 실행 |
| **4/23~26** | 심화교육 기능 + 기수 요약 리포트 | 매장 관리자 뷰 개선, 기수 요약을 AI 리포트 탭으로 통합 |

### 가장 어려웠던 것들

| 문제 | 상황 | 어떻게 해결했는가 |
|------|------|-----------------|
| **데이터가 다 흩어져 있다** | 시험은 구글 폼, 출결은 타임인아웃, 일지는 노션 — 전부 다른 곳 | 하나의 데이터베이스로 통합하고, 외부 데이터는 자동으로 가져오게 만들었다 |
| **노션 데이터 이관** | 기존 교육일지 82개를 새 시스템으로 옮겨야 함 | 자동 변환 도구를 만들어서 일괄 처리 |
| **시험 채점 로직이 복잡** | 구글 폼 시험의 채점 방식이 7가지 | 기존 시스템의 코드 267줄을 분석해서 호환되는 방식으로 구현 |
| **출결을 매일 수작업** | 매일 타임인아웃 접속 → 엑셀 다운 → 업로드 | 자동 로봇이 하루 4번 알아서 수집하게 자동화 |
| **"교육 효과 증명" 방법이 없음** | 교육이 효과 있었는지 데이터로 답할 수 없었음 | 8개 지표를 가중치로 합산하는 '적응지수' 설계 |
| **매장 인수인계가 어려움** | 교육 끝나고 매장에 보낼 때 교육생 정보 전달이 막막 | AI가 전 교육 데이터를 분석해 코칭 리포트를 자동 생성 → PDF로 전달 |
| **기수 간 비교 불가** | 1기 끝나고 2기 시작하면 비교할 기준이 없음 | 기수 요약 리포트(PDF) — 동일 양식으로 비교 가능 |

## 3. 시스템 전체 구성

### 사용자 3계층

```
┌─────────────────────────────────────────────────────────┐
│  교육 담당자 (나, 슈퍼관리자)                                │
│  → 전체 교육생 관리, 분석, AI 리포트, 설정                    │
├─────────────────────────────────────────────────────────┤
│  매장 관리자 / 교육 TF                                      │
│  → 배정된 교육생 현황 조회, R&P 평가 작성, 교육 총평           │
├─────────────────────────────────────────────────────────┤
│  교육생 (신입사원)                                          │
│  → 내 성적 확인, 교육일지 작성, 질문하기, 공지 확인             │
└─────────────────────────────────────────────────────────┘
```

### 자동화 현황

| 자동화 항목 | 주기 | 설명 |
|------------|------|------|
| 시험 결과 동기화 | 매일 09:30 | 구글 시트에서 자동으로 가져옴 |
| 출결 데이터 수집 | 하루 4회 | 타임인아웃에서 자동으로 가져옴 |
| 기수 아카이브 | 교육 완료 시 | 완료 기수를 읽기 전용으로 전환 |

## 4. 페이지별 기능 가이드

### 4.1 교육 담당자 페이지 (나만 보는 관리 화면)

#### 홈 대시보드
- **무엇**: 로그인하면 처음 보이는 화면. 오늘의 교육 현황을 한눈에 파악.
- **주요 내용**:
  - D-day 배지 (입문교육 몇 일차 / 종료까지 D-n)
  - 교육 일정 타임라인 (입문교육 → 심화교육 바)
  - 차시별 평균 점수 카드 + 추이 차트
  - 주의 교육생 알림 (위험/주의 자동 판별)
  - 교육일지 제출 현황 (오늘 누가 썼고 누가 안 썼는지)
  - 실습일지 실적 합계
  - 질문 관리 카드 (답변 대기 수 + 최근 질문)
  - 최근 공지사항
- **왜 필요**: 아침에 출근해서 이 화면만 보면 오늘 뭘 챙겨야 하는지 바로 파악 가능

#### 공지사항
- **무엇**: 교육생에게 전달할 공지를 작성/수정/삭제
- **주요 기능**:
  - 기수별로 공지 관리
  - 우선순위 3단계 (일반 / 중요 / 긴급)
  - 교육생이 로그인하면 안 본 공지를 팝업으로 자동 표시
- **왜 필요**: 카톡으로 공지하면 묻히는데, 시스템에서 강제 팝업으로 확실히 전달

#### 출결
- **무엇**: 교육생별 출결 현황을 날짜별로 확인
- **주요 기능**:
  - 타임인아웃 데이터 자동 수집 (하루 4번)
  - 날짜별 조회 + 교육생별 출석/지각/조퇴/결석 표시
  - 확인 필요한 교육생 자동 하이라이트
  - 엑셀 수동 업로드도 가능 (자동 수집 실패 시 백업)
  - 퇴사자는 반투명 처리 + 퇴사 뱃지 표시
- **왜 필요**: 매일 타임인아웃 들어가서 확인하는 시간 절약. 누가 자주 지각하는지 패턴 파악

#### 테스트 (시험 관리)
- **무엇**: 구글 폼 시험 결과를 자동 수집하고 분석
- **주요 기능**:
  - "차시별 성적" 탭: 차시별 평균/최고/최저/응시 인원 카드 + 학생별 성적 테이블
  - "시험 분석" 탭: 문항별 정답률 (낮은 순 정렬), 정답 수정 + 재채점, 서술형 수동 채점
  - 엑셀 다운로드 (결과 + 상세 로그 + 차시별)
  - 동기화 버튼 (수동) + 자동 동기화 (매일 9:30)
- **왜 필요**: 14차시 × 14명 = 196개 시험 결과를 엑셀로 관리하는 건 불가능. 자동 수집 + 시각화로 즉시 파악

#### 교육일지
- **무엇**: 교육생들이 매일 작성하는 교육일지를 모아서 확인
- **주요 기능**:
  - 날짜별 필터 — 오늘 누가 썼고, 누가 안 썼는지
  - STEP 1(배운 것) / STEP 2(어려운 것) / STEP 3(느낀 것) 구조
  - 참여 점수 자동 계산 (내용을 썼으면 1점, 3개 STEP 다 쓰면 3점)
  - 자신감 자기평가 (교육생이 선택)
  - 교육자 코멘트 (채팅형 — 코멘트 달면 교육생이 답글 가능)
  - 우수 학습 뱃지 (3/3 작성 시 ⭐ 표시)
  - 자율학습 노트 (보라색 테마, 하루 여러 개 가능)
- **왜 필요**: 노션에서 일지 하나하나 열어보는 것 vs 한 화면에서 제출 현황 + 내용 + 코멘트까지 관리

#### 실습일지
- **무엇**: 매장 실습 후 교육생이 작성하는 일지 + 실적 기록
- **주요 기능**:
  - 4개 텍스트 섹션: 기억에 남는 고객 / 선배 비법 / 칭찬 / 보완점
  - 4개 숫자 필드: 상담 건수 / 견적 건수 / 수주 건수 / 수주 금액
  - 날짜별 전체 실적 합계 표시
  - 제출 현황 (누가 썼고 누가 안 썼는지)
  - 교육생별 실적 뱃지
- **왜 필요**: 실습 현장에서의 경험 기록 + 초기 영업 실적을 추적

#### 질문 관리
- **무엇**: 교육생이 올린 질문에 답변하는 채팅형 Q&A
- **주요 기능**:
  - 탭 필터: 전체 / 답변 대기 / 답변 완료 / 보관됨
  - 채팅 UI (학생=왼쪽 회색, 교육자=오른쪽 파란 말풍선)
  - 대기 질문 수 빨간 뱃지
  - 수정/삭제 (자기 메시지만)
- **왜 필요**: 카톡으로 질문 받으면 다른 대화에 묻힘. 여기서는 질문별로 정리되고 상태 추적 가능

#### AI 리포트
- **무엇**: AI가 교육 데이터를 분석해서 교육생별 코칭 리포트를 생성
- **주요 기능**:
  - "AI 리포트" 탭:
    - 프롬프트 복사 버튼 (AI에게 분석을 맡길 때 사용)
    - 생성된 리포트 목록 (종합분석 / 분야별 분석)
    - 리포트 열람 (구조화된 카드형 — 피드백, 체크리스트, 오답 분석, 추천 교육 스타일)
    - 수정 기능 (AI가 생성한 내용을 교육자가 직접 수정 가능)
    - PDF 다운로드 (매장 관리자에게 전달용)
  - "기수 요약" 탭:
    - 기수 전체를 한 장으로 요약 (KPI 6개 + 카테고리별 성적 + 적응지수 분포)
    - PDF 다운로드 (기수 간 비교용)
- **왜 필요**: 14명분 리포트를 직접 쓰면 한 명당 2~3시간. AI가 데이터를 분석해서 초안을 만들어주면 확인만 하면 된다. 기수 요약은 2기 시작할 때 1기와 비교하는 용도.

#### 교육생 분석
- **무엇**: 교육생 개인별 상세 분석 페이지
- **주요 기능**:
  - 프로필 + 성적 추이 차트
  - 카테고리별 정답률 (어떤 분야가 약한지)
  - 출결 히트맵
  - 태그 추적 (극복한 약점 / 새로운 약점 / 고질적 약점)
  - 교육자 메모 타임라인 (수업태도, 상담, 칭찬, 주의, 일반 5가지 카테고리)
  - 위험도 자동 판별 (6가지 신호)
  - 적응지수 표시
- **왜 필요**: "이 교육생 어때요?"에 감이 아니라 데이터로 답하기 위해

#### 교육 설문
- **무엇**: 자기효능감 등 교육 관련 설문 조사 관리
- **왜 필요**: 교육 전후 자기효능감 변화 측정 → 교육 효과의 심리적 지표

#### 교육 인사이트
- **무엇**: AI가 시험 데이터를 분석해서 교육 방향을 제안
- **주요 기능**: 취약 영역 분석 + 교육 추천 + 변형 문제 제안
- **왜 필요**: "어디를 더 가르쳐야 하지?" 에 대한 데이터 기반 답변

#### 기수 관리 (설정)
- **무엇**: 기수 + 교육생 정보 관리
- **주요 기능**:
  - 기수 추가/수정 (입문교육 일정, 심화교육 일정, 구글 시트 연결)
  - 교육생 추가/수정
  - 퇴사 처리 (퇴사일 + 사유 기록 → 전 페이지에서 반투명 처리)
  - 비밀번호 초기화
  - 기수 아카이브 (교육 완료 → 읽기 전용 전환)
- **왜 필요**: 2기, 3기가 시작되면 기수별로 데이터를 분리 관리해야 한다

#### 심화교육 종합
- **무엇**: 심화교육(매장 배치 후) 전체 현황을 한눈에
- **주요 기능**: R&P 평가 현황, 벤치마킹 현황, 최종 총평 현황, 교육생별 진행 상태
- **왜 필요**: 심화교육 기간에 여러 매장에 흩어진 교육생들의 상태를 중앙에서 파악

### 4.2 매장 관리자 페이지 (매장 교육 담당자가 보는 화면)

#### 매장관리자 홈
- **무엇**: 배정된 교육생의 교육 성장 추이를 한눈에 확인
- **왜 필요**: 매장 관리자가 교육생의 입문교육 배경을 빠르게 파악

#### R&P 평가
- **무엇**: 주차별 역할 수행(Role Play) 평가 작성
- **주요 기능**: 강점 태그 / 개선 태그 선택 + 코멘트 + R&P 영역 선택
- **왜 필요**: 입문교육 → 심화교육에서 실전 역량이 어떻게 변하는지 추적

#### 교육 총평
- **무엇**: 심화교육 종료 시 최종 평가
- **주요 기능**: 매장적합도 / 독립수행도 / 고객응대 / 제품지식 각 5점 척도 + 종합 의견
- **왜 필요**: 최종적으로 "이 교육생이 매장에 적합한가"를 구조화된 평가로 남기기

#### 리포트 열람
- **무엇**: 교육 담당자가 만든 AI 코칭 리포트를 열람
- **왜 필요**: 매장 관리자가 교육생의 입문교육 기간 분석을 한눈에 파악

#### 교육생 목록 / 상세
- **무엇**: 배정된 교육생 정보 확인 (읽기 전용)
- **왜 필요**: 교육생의 시험 성적, 오답 패턴 등을 매장에서도 확인 가능

#### 시험 점수
- **무엇**: 심화교육 기간 시험 점수 확인
- **왜 필요**: 심화교육 시험 결과를 매장 관리자도 확인

### 4.3 교육생 페이지 (신입사원이 보는 화면)

#### 교육생 홈
- **무엇**: 로그인하면 보이는 첫 화면
- **주요 기능**: 안 본 공지 팝업 + 내 최근 성적 + 출결 현황
- **왜 필요**: 교육생이 자기 상태를 스스로 확인

#### 공지사항
- **무엇**: 교육 담당자가 올린 공지 전체 모아보기
- **주요 기능**: 아코디언 펼침 + NEW 뱃지 (안 본 공지)
- **왜 필요**: 팝업에서 놓친 공지도 다시 확인 가능

#### 출결
- **무엇**: 내 출결 기록 확인
- **왜 필요**: 본인의 출석/지각/결석 현황 자기 확인

#### 테스트
- **무엇**: 내 시험 성적 확인 + 문항별 답안 비교
- **주요 기능**: 차시별 점수 + 내가 쓴 답 vs 정답 비교
- **왜 필요**: 어디서 틀렸는지 스스로 복습

#### 교육일지
- **무엇**: 매일 교육일지 작성
- **주요 기능**:
  - STEP 1(오늘 배운 것) / STEP 2(어려웠던 것) / STEP 3(느낀 것) 작성
  - 자신감 자기평가 선택
  - 교육자 코멘트 확인 + 답글
  - 자율학습 노트 (별도 모드, 하루 여러 개 가능)
  - 날짜 선택 (최근 7일 중 선택)
- **왜 필요**: 매일의 학습을 구조화해서 기록 → 교육 담당자가 참여도 파악

#### 실습일지
- **무엇**: 매장 실습 후 일지 + 실적 기록
- **왜 필요**: 현장 경험 기록 + 초기 영업 실적 추적

#### 질문하기
- **무엇**: 교육 담당자에게 질문 올리기 (채팅형)
- **주요 기능**: 질문 목록 + 채팅 화면 / 새 질문 / 답글 / 수정 / 삭제
- **왜 필요**: 수업 중 못 물어본 것, 복습하다 궁금한 것을 비동기로 질문

#### 심화교육 (벤치마킹)
- **무엇**: 매장 배치 후 선배 관찰 기록 작성
- **왜 필요**: 선배의 영업 노하우를 구조화해서 기록

#### 설문
- **무엇**: 교육 관련 설문 응답 (자기효능감 등)
- **왜 필요**: 교육 전후 심리적 변화 측정

#### 심화 시험
- **무엇**: 심화교육 기간 시험 응시 결과 확인
- **왜 필요**: 심화교육 시험 결과 자기 확인

## 6. 교육 효과 측정 전략

### 핵심 질문: "교육이 정말 효과가 있었나?"

이걸 답하려면 **입문교육에서 잘한 교육생이 매장에서도 잘하는지** 추적해야 한다.

### 3단계 추적

```
입문교육 (~4/22)        심화교육 (5/4~)         매장 실전 (배치 후)
━━━━━━━━━━━━━━━       ━━━━━━━━━━━━━━━       ━━━━━━━━━━━━━━━
시험 성적               R&P 평가               상담 건수
교육일지 참여            실습일지 실적            수주 건수/금액
출결                   매장 적응도              상담→수주 전환율
질문 활동               벤치마킹                수주 성장률

  📚 학습력       →      🔧 적용력        →     📈 성과력
```

### 적응지수 — 교육생 종합 점수

교육생의 전체적인 교육 적응도를 **하나의 숫자**로 표현:

| 평가 항목 | 비중 | 의미 |
|----------|------|------|
| 시험 평균 점수 | 30% | 지식을 얼마나 습득했는가 |
| 약점 분야 비율 | 15% | 특정 분야에 구멍이 있는가 |
| 출석률 | 13% | 성실하게 참여했는가 |
| 교육일지 참여도 | 15% | 매일 학습을 정리했는가 |
| 성장 기울기 | 10% | 시간이 갈수록 나아지고 있는가 |
| 자신감 추이 | 8% | 심리적으로 성장하고 있는가 |
| 만성 오답 비율 | 5% | 같은 실수를 반복하는가 |
| 교육자 메모 톤 | 4% | 교육자가 긍정적으로 보는가 |

**그룹 분류**: 상위(75점+) / 중위(50~74점) / 하위(50점 미만)

### 교차 분석 — 수주 데이터가 쌓이면 답할 수 있는 질문들

| 질문 | 기대 인사이트 |
|------|-------------|
| 시험 잘 본 교육생이 수주도 잘하나? | 시험이 실전을 예측하는지 검증 |
| 침대 잘 아는 사람이 침대를 많이 파나? | 카테고리별 교육 시간 조정 근거 |
| 교육일지 열심히 쓴 사람이 성과도 좋나? | 교육일지의 실효성 검증 |
| 적응지수 높은 사람이 전환율도 높나? | **가장 중요** — 2기 위험 교육생 조기 발견 기준 |

### 주차별 리포트 계획

| 시점 | 리포트 | 상태 |

## 7. 회사에 어떻게 말할 것인가

### 한 문장 요약

> "비개발자인 교육 담당자가 AI 도구를 활용해 1개월 만에 교육 관리 시스템을 직접 구축했습니다. 기수당 50시간 이상의 반복 업무를 자동화하고, 교육 효과를 데이터로 증명할 수 있는 체계를 마련했습니다."

### 업무 효율화 (숫자로)

| 기존 | 현재 | 절감 |
|------|------|------|
| 시험 채점 + 정리: 30분~1시간/차시 | 자동 (0분) | 차시당 ~1시간 |
| 출결 확인: 10분/일 | 자동 (0분) | 월 ~3.5시간 |
| 교육일지 취합: 20분/일 | 자동 (0분) | 월 ~7시간 |
| 개인별 피드백 작성: 1시간/명 | AI 생성 (5분/명) | 14명 기준 ~13시간 |
| 리포트 작성: 2~3시간/명 | AI + PDF (10분/명) | 14명 기준 ~30시간 |
| **합계** | | **기수당 50시간+ 절감** |

### 핵심 메시지 3가지

**1. 비용 절감**
- 외주 개발 시 최소 2,000~5,000만 원 → AI 도구 월 구독료로 구축
- 기수당 50시간+ 반복 업무 자동화

**2. 교육의 가치를 숫자로 증명**
- "교육 잘했습니다"가 아니라, "적응지수 상위 그룹의 수주 전환율이 2배 높습니다"
- 기수 간 비교로 교육 개선 효과를 정량화

**3. AI 활용 내재화**
- 교육 담당자가 직접 AI로 시스템 구축 → 외부 의존 없이 지속 운영
- 현장을 가장 잘 아는 사람이 만들었기 때문에, 현장에 딱 맞는 시스템
- AI 시대에 비개발자도 직접 업무 도구를 만들 수 있다는 사례

### 최종 산출물 계획

1기 교육이 완전히 끝나면 만들 수 있는 것:

| 산출물 | 누구에게 | 내용 |
|--------|---------|------|
| 1기 교육 종합 보고서 | 경영진/인사팀 | 요약 1장 + 상세 (교육 효과 데이터 증명) |
| 예측 변수 랭킹표 | 교육 팀 | 어떤 교육 지표가 매장 성과를 가장 잘 예측하는가 |
| 카테고리별 교육 효과 분석 | 교육 팀 | 대분류별 정답률 vs 수주 비중 → 교육 시간 조정 근거 |
| 위험 교육생 조기 발견 기준 | 교육 팀 | 1기 데이터로 만든 기준 → 2기에서 조기 개입 |
| 2기 교육 개선 제안서 | 교육 팀/경영진 | 1기 데이터 기반 구체적 개선안 |
