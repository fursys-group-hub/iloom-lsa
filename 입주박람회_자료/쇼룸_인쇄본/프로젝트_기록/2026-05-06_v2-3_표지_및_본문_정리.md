### 9. PDF 인쇄 버튼 이슈 진단

증상: 우측 상단 "PDF로 저장 / 인쇄" 버튼 클릭해도 인쇄 다이얼로그 안 뜸. 키보드 `Ctrl+P`는 정상 작동.

진단 과정:
- `type="button"` 추가, z-index 100 → 9999, addEventListener 백업
- 콘솔 로그 심어서 클릭 이벤트 확인 → **클릭은 정상으로 잡힘**
- 진짜 원인 = Chromium의 file:// URL 보안 정책: `'file:' URLs are treated as unique security origins.` 에러로 페이지 내부 `window.print()` 차단 (Ctrl+P는 브라우저 단축키라 우회됨 — 알려진 Chromium 이슈)

대응:
- `window.top.print()` 시도 후 실패 시 `window.print()` 폴백
- 그래도 안 되면 alert로 "Ctrl+P 직접 누르세요" 안내

→ 근본 해결은 로컬 서버(http://) 사용이지만, 영업 현장에서는 그냥 Ctrl+P 단축키 안내가 현실적.

### 10. 폴더 구조 개편 — `쇼룸_인쇄본/` 분리

수지님이 직접 정리:
```
입주박람회_자료/
├── 사용 이미지본/   (이미지 — 그대로)
├── 쇼룸_웹/         (웹 버전)
└── 쇼룸_인쇄본/     ← HTML들 + fonts/ 모두 이쪽으로 이동
    ├── 키큰옷장_한샘비교_v1~v2-3.html
    ├── fonts/
    └── 프로젝트_기록/
```

상대경로 깨짐 처리:
- `src="사용 이미지본/..."` → **`src="../사용 이미지본/..."`** 일괄 변경 (15곳)
- `url('fonts/...')` → 처음에 `../fonts/`로 잘못 바꿨다가 → **`fonts/`** 로 원복 (fonts 폴더는 쇼룸_인쇄본 안에 같이 옮겨져 있어서 원래 경로가 맞음)

## 🤔 결정한 것

### 메시지 톤
- 표지에서 "동등 견적" 같은 강한 단정 표현 빼고, 자료 제목("타사 VS 컬렉트 키큰옷장 / 스펙 및 가격 비교")으로 톤 다운
- 핵심 메시지(비슷한 금액, 더 좋은 마감)는 Page 2 검은 박스 한 곳에 집중

### 시각 강조 도구
- **빨간 주의 박스 (#D32F2F)** — 시공 관련 경고 메시지에만 한정 사용 (남발 금지)
- 그 외 정보는 흑백 미니멀 톤 유지

### 폴더 구조
- `쇼룸_웹` (디지털 키오스크용 / 인터랙티브) 와 `쇼룸_인쇄본` (인쇄용 PDF 출력) 분리 → 같은 자료라도 매체별 최적화 가능
- 인쇄본은 fonts까지 자체 폴더에 두고 자족적으로 운영
