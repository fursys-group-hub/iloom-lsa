### 2. Slide 8 가격 카드에 옷장 모듈 시각화 신규 추가

수지님 요청: "인쇄본처럼 구성을 직접 보여주는 건 어때? 이런 느낌으로!"

**구현 흐름**:
- CSS — `.price-visual-modules`, `.price-visual-side`(서라운딩 띠), `.price-visual-module`, `.price-visual-summary` 등 신규
- HTML — 가격 카드 안에 visual 영역 추가
- 데이터 — `PRICE_DATA` 6개 항목 모두에 `visual` 필드 추가:
  - `surround` (서라운딩 양옆 띠 표시 여부) — 붙박이장만 true
  - `modules` (이름 + 폭 + 이미지 경로)
  - `totalWidth` / `widthNote` / `suitable` / `suitableNote`
- JS — `renderPrice()`에 동적 렌더 로직 추가 (좌우 화살표 누를 때마다 visual도 다시 그림)

**사용한 모듈 이미지**: 기본장.png / 멀티선반 하프장.png / 긴장.jpg / 2단 속서랍 옷장.jpg / 3단 서랍 옷장.jpg

### 3. 레이아웃 좌우 2단으로 재구성

처음엔 세로 3단 (head → 모듈+요약 가로 → 가격 가로 2열) 으로 짰는데, 모듈 이미지가 작고 가격 영역이 아래로 밀려서 잘림. 수지님 피드백: "왼쪽엔 이미지들 오른쪽엔 그와 관련된 총 폭, 적합 평형대 그리고 가격 비교 이런식으로 놔두는 건?"

**변경 전 → 후**:
```
[ head: 구성 / 페이저 ]                [ head ]
[ 모듈 시각화 | 사양 ]      →         ┌─────────┬─────────┐
[ 타사  |  컬렉트 ] (가로 2열)         │  모듈    │  사양   │
                                      │  시각화   │ ──────  │
                                      │  (꽉찬)   │  타사   │
                                      │          │ ──────  │
                                      │          │  컬렉트  │
                                      └─────────┴─────────┘
```

- `.price-body` 2열 그리드 (1.1fr 1fr)
- `.price-body-left` — 모듈 시각화 전용 (회색 배경 카드)
- `.price-body-right` — 사양 박스 + 가격 세로 스택
- `.price-vs` — 기존 가로 2열 → 세로 flex column로 변경
- `.price-vs .price-side` — 가격 폰트 50px → 36px (오른쪽 컬럼 폭에 맞춰 컴팩트)

### 4. 모듈 이미지 단계적으로 키움

수지님 피드백 두 차례:
- 1차 ("옷장이 좀 더 크게 보였으면"): 180px → **280px**
- 2차 ("좀 더 키워도 될 것 같아"): 280px → **320px** → 좌우 2단 재구성 후 **420px**
- 라벨 글씨도 비례해서 키움 (모듈명 14→18px, 폭 12→15px, 서라운딩 12→16px)

### 5. 사양 박스(총 폭/적합 평형) 글씨 키움

수지님 피드백: "글씨들을 키울 수 있을까?"
- 라벨: 11px → **14px**
- 수치: 18px → **26px**
- 부가 설명: 12px → **15px**
- 패딩 14→18px / gap 18→22px 도 같이 늘림

### 7. PDF 인쇄 버튼 이슈 마무리

수지님이 모르고 옛 폴더 위치(`입주박람회_자료/키큰옷장_한샘비교_v2-3.html`)의 파일을 봤는데, 그건 이미 `쇼룸_인쇄본/`으로 이동된 상태라 폰트/이미지 경로 안 맞아서 폰트 깨짐. 폰트 경로 `../fonts/` → `fonts/` 로 원복 (fonts 폴더는 쇼룸_인쇄본 안에 같이 옮겨져 있었음).

`window.print()` 차단 이슈는 Chromium의 `'file:' URLs are treated as unique security origins` 정책 때문 — 키오스크에서는 어차피 인쇄가 아니라 화면 표시용이라 영향 없음. 인쇄본은 Ctrl+P 단축키 안내로 우회.

## 🤔 결정한 것

### 인쇄본 ≠ 웹쇼룸 — 매체별 차별화는 인정
- 표지: 인쇄본은 "타사 VS 컬렉트 키큰옷장 / 스펙 및 가격 비교"만 → 웹은 같은 카피 + CTA 버튼 + 터치 안내 (키오스크 운영상 필수 요소만 추가)
- 가격 차이: 인쇄본은 검은 박스에 "비슷한 금액..." 메시지 → 웹은 검은 박스 통째로 삭제 (인터랙티브 슬라이드라 카드별로 명확히 비교됨)

### 가격 카드 레이아웃 (좌우 2단)
- 좌: 모듈 시각화에 화면 공간 충분히 할애 (옷장이 영업의 핵심 시각 자료)
- 우: 사양 + 가격 모두 묶어서 컴팩트하게 (스크롤 없이 한눈에)

### 배포 도구 선택
- 처음에 Railway 고려 → 정적 HTML에는 과한 도구 (서버 세팅 필요) → Netlify Drop 추천
- Netlify Drop = 회원가입 없이 폴더 드래그만 = 5분 완료
- 한글 폴더명은 URL에서 `%E...` 인코딩되지만 작동에는 문제없음
