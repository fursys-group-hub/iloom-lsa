# 2026-05-06 — 박람회 키오스크 v1 초기 구현

기존 인쇄용 PDF 자료(`키큰옷장_한샘비교_v2-3.html`)를 가로형 TV용 인터랙티브 키오스크로 변환한 첫 번째 버전.

---

## ✅ 오늘 한 것

### 1. 새 프로젝트 폴더 + 기획서
- `입주박람회_자료/쇼룸_웹/` 폴더 신설
- `기획서.md` — 8개 슬라이드 구성, 자동재생/터치 동작, 키보드 단축키 등 정리
- `프로젝트_기록/` 폴더 + `프로젝트_로그.md` 인덱스
- 인쇄용 v2-3 HTML은 그대로 두고, 화면용으로만 별도 분리 (인쇄/화면 둘 다 살아있음)

### 2. 메인 HTML 한 파일로 구현 (`index.html`)

**8개 슬라이드:**
1. **표지** — 큰 타이틀 + "비교 시작" CTA + 손짓 힌트
2. **한눈에 비교** — 7행 4컬럼 비교표 + "350원 차이" 결론 박스 (검정 배경)
3. **표면재** — LPM vs 코팅페이퍼 2단 카드, **클릭 → 확대 모달**
4. **두께** — 18.5/15.5 vs 18/15mm 애니메이션 바 차트 (slide active 시 가로로 차오름)
5. **부품** — Hettich 50,000회 / Salice 200,000회 / 옷걸이봉 — 3장 카드, **클릭 → 확대 모달**
6. **모듈 + 시뮬레이터** — 좌측 모듈 6종 리스트, 우측 옷장 시뮬레이터 (모듈 클릭 → 추가, 클릭 → 제거, 총 폭/적합 평형 자동 계산)
7. **도어** — 리디 vs 스톤W 2단 카드, **클릭 → 확대 모달**
8. **가격 슬라이더** — 붙박이장/옷장 탭 + 8개 구성 좌우 화살표/점 네비, 한샘 vs 컬렉트 가격 즉시 비교, **차이 계산 자동**

### 3. 키오스크 동작 시스템

**자동재생 (Attract 모드)**:
- 슬라이드당 8.5초, 끝까지 가면 자동으로 표지로 순환
- 하단 진행 바가 채워지면서 다음 슬라이드 예고 (진행감)

**인터랙티브 모드**:
- 손님이 화면 터치 → 자동재생 멈추진 않음(계속 진행), 단 30초 무동작이면 표지로 복귀
- 좌우 스와이프, 좌우 화살표 키, 하단 점 클릭, 좌우 네비 버튼 — 4가지 방식 모두 지원
- 모달 열리면 자동재생 일시정지, 닫으면 재개

**제스처/단축키**:
- 좌우 스와이프 (80px 이상)
- ←/→ 키, Space (일시정지), Esc (모달 닫기)
- F11 — 브라우저 풀스크린

### 4. 인터랙티브 컴포넌트

**확대 모달**: 표면재 2개, 부품 3개, 도어 2개 — 총 **7개 항목** 클릭 시 큰 사진 + 카탈로그 톤 설명문 + 핵심 숫자 2개 + bullet 강점 3개 표시

**모듈 시뮬레이터**:
- 6종 모듈 카드 → 클릭으로 우측 옷장에 추가
- 추가된 모듈은 시각적으로 "차곡차곡" 쌓이며 폭에 따라 너비 비율 다르게 표시(900폭=90px, 450폭=50px)
- 모듈 클릭 → 제거 (×버튼 호버 시 표시)
- 총 폭(서라운딩 +250mm 자동 가산), 모듈 개수, 적합 평형(드레스룸/안방/30평형 등) 자동 계산
- 최대 8개 제한

**가격 슬라이더**:
- 붙박이장 4개 + 옷장 4개 = 총 8개 구성
- 각 구성마다 컬렉트(리디 + 스톤W) vs 타사(가성비/프리미엄 라인) 가격 비교
- "직접 비교 모델 없음" 케이스도 자연스럽게 처리 (— 표시 + 차별화 구성 메시지)
- 차이가 5만 원 미만이면 "사실상 같은 가격" 메시지, 컬렉트 우위면 "컬렉트가 N원 더 저렴" 메시지, 타사 우위면 "학생방 패키지로 객단가 보강" 메시지 (영업 가이드 톤 자동 전환)

### 5. 디자인 — 인쇄용 톤 그대로 화면용으로 확장

- **폰트**: Paybooc 4종 weight 그대로 사용 (`../fonts/`)
- **컬러**: 검정 #2A2A2E, 컬렉트 그레이지 #F7F5F1 유지 + 골드 #B89968 포인트 (영상감 추가)
- **타이포 위계**: 표지 96pt → 슬라이드 제목 56pt → 본문 22pt (TV 멀리서 봐도 보이게 인쇄용보다 4~5배 키움)
- **카드 디자인**: 모서리 16~24px, 호버 시 부드러운 lift + 그림자
- **자동 스케일**: `transform: scale()`로 1920×1080 stage가 어떤 화면 크기에도 비율 유지하며 맞춤 (16:9가 아닌 화면도 안 깨짐)

### 6. 이미지/폰트 — 부모 폴더 재사용

새 폴더에 자산 중복 안 만들고 기존 `../사용 이미지본/`, `../fonts/` 그대로 참조. 외부 호스팅(`https://iloomproduct.fursys.com/...`)도 그대로 사용 (긴장/속서랍/서랍/하부선반 모듈 4종 — 인터넷 필요).

## 🤔 결정한 것

### 포지셔닝
- **인쇄용 v2-3 HTML은 건드리지 않음** — 영업이 손님에게 PDF로 출력해서 줄 때 그대로 사용
- **쇼룸용은 별도 폴더** — 박람회 부스 TV에서만 사용, 인쇄와 분리

### 구성/순서
- 부록(영업 시나리오 Q&A 4개) **제외** — 영업 내부용이라 손님 화면에 안 띄움
- 메인 가격표 8개 행 → 슬라이더로 변환 (한 번에 다 보여주기보단 손님이 자기 구성 클릭하며 비교하게)
- 8개 슬라이드 단일 파일 — 한 파일 더블클릭으로 끝(외부 의존 최소화, 박람회 현장 간편)

### 자동재생 vs 인터랙션 균형
- **터치해도 자동재생 안 멈춤** (Pause 안 시킴) → 손님이 잠깐 만지고 가도 다음 슬라이드로 자연스럽게 진행
- 모달이 열려있을 때만 일시정지 (모달 본 손님은 그 슬라이드 더 보고 싶다는 신호)
- 30초 무동작 → 표지 복귀로 다음 손님 맞이

### 영업 시나리오 메시지 자동 전환
- 1통반(D행)은 한샘이 더 저렴한 케이스 — Q&A에서 "학생방 패키지 객단가 보강" 멘트가 영업 가이드였음
- 가격 슬라이더에서 이 케이스에 도달하면 자동으로 그 메시지 띄움 → 영업이 옆에서 안 봐도 메시지 일관성 유지
