## ✅ 오늘 한 것 (큰 카테고리별로)

### A. 키오스크 인터랙션 개선

**v1.3 — 손님 터치 시 자동재생 멈춤**
- 처음엔 터치해도 자동재생이 계속 진행돼서 손님이 모듈 시뮬레이터 만지는 중에도 슬라이드가 넘어감 → 박람회 키오스크 표준 동작으로 수정
- 손님이 화면 클릭/터치하면 → 자동재생 즉시 정지 + 우상단 "● 자동재생 일시정지" 빨간 인디케이터 표시
- 30초 무동작 시 → 표지로 자동 복귀 + 재생 시작
- 마우스 이동(`mousemove`)은 자동재생 멈추지 않음 (직원이 부스 앞 지나가도 영향 X)
- 모달 열림 → 자동재생 정지

**v1.7 — 네비 아이콘 SVG 교체 + 버튼 크기 조정**
- ‹ › 화살표 문자가 Paybooc 폰트에서 안 보이던 문제
- 모든 화살표를 SVG chevron으로 교체 (어떤 폰트에서도 보장)
- 메인 navbar 화살표 56 → 42px, 가격 슬라이더 80 → 64px (좌측 텍스트/점들과 비율 균형)

### C. 콘텐츠 정리 (영업사원 톤 → 객관적 톤)

**v1.7 — "입주박람회" 표현 제거**
- 네비 브랜드명: "iloom COLLECT 키큰옷장 · 입주박람회" → "iloom COLLECT 키큰옷장"
- 표지 부제: "입주박람회 한정 · 한 단계 위 내구성을 같은 가격에" → "한 단계 위 내구성을 같은 가격에"
- 박람회뿐만 아니라 매장 상시 키오스크로도 자연스럽게 사용 가능

**v1.9 — 한눈에 결론에서 350원/견적 언급 제거**
- 기존: "350원 — 사실상 동등 견적인데, Hettich · Salice · LPM 일체 마감이 기본"
- 수정: "타사가 카탈로그에 명시조차 안 하는 Hettich · Salice · LPM 일체 마감이 컬렉트 키큰옷장에는 모두 기본으로 들어갑니다"
- 이유: 슬라이드 2(한눈에) → 슬라이드 8(가격) 흐름에서 가격을 슬라이드 8까지 미루고 슬라이드 2는 사양만 강조

**v1.12 — 모달 어투 통일 + em dash 제거 + ✓ 글머리**
- 영업사원 어투("~해요/이에요/돼요") → 객관적 톤("~합니다/입니다") 통일
- 표면재/부품/도어 모달 7개 desc + pills 모두 정리
- em dash(—)가 폰트 렌더링 문제로 안 보이던 이슈 → 콜론(:) 또는 자연스러운 한국어로 변환
  - "강철 슬라이드 + 볼베어링 — 5만 회 테스트 검증" → "강철 슬라이드 + 볼베어링: 5만 회 개폐 테스트 검증"
  - "뒷판·서랍재까지 LPM 일체 — 부위별 마감 차이 없음" → "뒷판·서랍재까지 LPM 일체로 부위별 마감 차이 없음"
- modal-pills li에 골드 ✓ 글머리 추가 (시각적 항목 구분 명확)

### D. 카드 디자인 일관성

**v1.10/v1.11 — 마소재 슬라이드 정리**
- zoom-hint(우상단)와 카드 tag("5년이면 들뜸"/"10년 후 그대로")가 겹치던 이슈 → zoom-hint를 사진 안 → 카드 우하단으로 이동
- 사진 height 200 → 280px, 본문 불릿 16 → 20px 시원하게
- 카드 padding-bottom 24 → 44 (zoom-hint 자리 마련)

**v1.16 → v1.17 — 도어 카드를 표면재 카드 디자인으로 통일**
- 처음에 부품 슬라이드 디자인을 표면재/도어에 적용했다가, 수지님이 "표면재처럼 도어 카드를 변경"이라고 정정 → 표면재 디자인을 기준으로 재정렬
- 도어 카드 head 구조: `[이름 + 부제]` 좌측 / `[색상 칩]` 우측 (justify-content: space-between)
- zoom-hint 우상단 → 우하단으로 이동, 일반 bullet 리스트 + 우하단 "🔍 자세히 보기"로 통일
- 부품 카드(슬라이드 5)는 별도 디자인(번호 동그라미 + stat 박스 + ✓ 체크 리스트)로 그대로 유지

### E. 모듈 시뮬레이터 정리

**v1.2 — 새 모듈 이미지 6종 적용**
- 수지님이 6종 모듈 jpg 새로 만들어 `사용 이미지본/`에 추가 → 외부 fursys URL 의존성 모두 제거
- 박람회 와이파이 안 좋아도 100% 동작 가능 (USB로 노트북에 들고 가서 그대로 띄우기 OK)

**v1.4 — 옷장/붙박이장 토글 + 사진 입면 높이 통일**
- "서라운딩은 붙박이장일 때만" 정정 → 시뮬레이터 상단에 [옷장][붙박이장] 알약 토글 추가
- 옷장 모드: 서라운딩 없음, 총 폭 = 모듈 합산
- 붙박이장 모드: 양 끝 서라운딩 자동 추가 + 총 폭 +250mm
- `object-fit: contain` → `cover`로 모듈 사진 높이 통일 (위쪽 빈 공간 제거)

**v1.5 — 멀티선반 하프장 2줄 라벨 영역 고정**
- 긴 모듈명("멀티선반 하프장")이 좁은 컨테이너에서 2줄로 줄바꿈되어 사진 영역이 모듈마다 다르게 보이던 이슈
- `.sim-placed-name` height 38px 고정 + `flex-shrink: 0` → 1줄/2줄 모두 동일 라벨 영역 → 사진 영역 통일

**v1.6 — 서라운딩 박스 높이 제한 + 모서리 통일**
- 서라운딩이 라벨 영역까지 침범하던 이슈
- `.sim-side` 구조 변경: 박스(`.sim-side-box`) + ::after 빈 공간 (라벨 자리) → 사진 박스 높이까지만 차지
- 첫/마지막 자식만 둥근 모서리 (sim-side든 sim-placed든) → 한 옷장처럼 자연스럽게

**라벨 흰 박스 제거**
- 모듈 박스 안에 사진 + 라벨이 같이 있어 라벨 영역에도 흰 배경이 보이던 이슈
- `.sim-placed`는 투명 컨테이너로, 사진 영역(`.sim-placed-img`)만 흰 배경 + 보더
- 라벨은 박스 밖에 떠있는 텍스트로 자연스러움

### F. 모달 디테일

**v1.12 — 모달 제목 사이즈 조정**
- 44px → 32px (긴 제목 "LPM 일체 (저압 멜라민)"이 한 줄에 들어가도록)
- 추후 v1.14에서 다시 위계 조정

**v1.14 — 모달 폰트 위계 재정리**
- 알루미늄 같은 제품명이 더 강조되도록 위계 명확화
- 카테고리 14 → 18, 제목 32 → 40, 통계 숫자 28 (그대로), 본문 16 → 18, 체크 리스트 14
- 카테고리와 제목을 `.modal-header` wrapper로 묶어서 6px 가까이 + 그룹과 통계는 28px 떨어지게
- 줄 간격 1.6 통일, 본문만 1.8로 더 넓게 (한국어 가독성)

**모달 크기/사진 비율 통일**
- 사진 비율에 따라 모달 크기가 들쭉날쭉하던 이슈
- 모달 크기 1500×820px 고정 (작은 화면 95vw/90vh로 자동 축소)
- 사진 영역 좌측 가용 영역 가득, 본문 우측 520px 고정
- 사진은 기본 `cover` (가득 채움 + 비율 유지) — 부품/도어
- 표면재 단면 사진(`material-paper`/`material-lpm`)만 `fit: 'contain'` 옵션 추가 → 단면 디테일 잘리지 않게 전체 보임

## 🤔 결정한 것

### 디자인 원칙
- **검은 결론 박스 5개** (cmp-conclusion / pitch-quote / parts-impact / sim-summary / price-diff-row) → `.dark-block` 공용 톤
- **흰 카드 5+종** → `.surface-card` 공용 (변형: `.is-clickable` / `.is-iloom`)
- **iloom 강조 카드 외곽선** → 옅은 골드 `rgba(184,153,104,0.4)`
- **모든 클릭 가능한 카드 zoom-hint** → "🔍 자세히 보기" 단일 표현으로 통일

### 콘텐츠 톤
- **고객 화면**: 객관적/제품 관점 어투 ("~합니다/입니다")
- **영업 시나리오**(인쇄용 v2-3 부록 9~10): 영업사원 멘트 그대로 유지 (내부 교육용이라 화면용 별도 적용 안 함)
- **가격 언급**: 슬라이드 8 가격 슬라이더에서만 등장. 슬라이드 2~7은 사양 메시지만

### 카드 디자인 두 패턴
1. **부품 카드 패턴** (슬라이드 5 부품): 번호 동그라미 + 카테고리 + 이름 / stat 박스 / ✓ 체크 리스트
2. **표면재 카드 패턴** (슬라이드 3 표면재 / 슬라이드 7 도어): 이름 + tag (또는 색상 칩) / 일반 bullet 리스트 / 우하단 zoom-hint
- 두 패턴 공존 — 컨텐츠 성격에 맞게 선택

### 모듈 시뮬레이터 모드
